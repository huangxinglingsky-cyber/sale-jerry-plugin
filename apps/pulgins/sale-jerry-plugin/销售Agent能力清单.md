

## 🎯 完整能力清单

### 一、销售管理技能 (Sale-Jerry-Plugin)

| 序号 | 技能名称 | 对应 Skill | 核心能力 | 使用指令示例 |
| :---: | :--- | :--- | :--- | :--- |
| **01** | **项目初始化** | `sale-jerry-plugin:project-init` | 为客户项目创建标准化目录结构（9个标准目录）并生成核心模板文件，支持智能项目名称处理 | "初始化客户项目'农夫山泉-CMDB项目'" |
| **02** | **项目干系人录入** | `sale-jerry-plugin:stakeholder-input` | 智能识别文本/图片中的姓名、电话、职位、邮箱，自动更新到项目通讯录文件，支持追加/替换/智能合并模式 | "直接截图名片，或输入：张三 13800138000 运维总监" |
| **03** | **项目状态更新** | `sale-jerry-plugin:project-status-updater` | 安全更新项目状态文件的特定章节（如当前阶段、下一步计划），支持版本控制和冲突检测 | "更新项目进度为'POC阶段'" |
| **04** | **企业信息调研** | `sale-jerry-plugin:company-research` | 扮演专业调研分析师，解析企业名称补齐全称，智能分析所属行业，提供真实企业背景信息 | "调研'招商银行'的企业信息" |
| **05** | **同行案例匹配** | `sale-jerry-plugin:case-matching` | 从843个案例库中基于行业、模块、时间等多维度智能筛选最相关的成功案例，支持行业回退匹配、模块同义词匹配 | "匹配金融-银行行业的CMDB案例" |
| **06** | **销售准备一站式** | `sale-jerry-plugin:sales-prep` | 一站式完成企业调研、案例匹配和话术生成的完整流程，自动生成销售准备资料包 | "为'中国银行'准备销售资料" |
| **07** | **销售话术生成** | `sale-jerry-plugin:sales-script` | 扮演20年经验的顶级销售专家，针对运维部门生成场景化开场、痛点挖掘、价值呈现、异议处理等完整销售话术 | "生成针对运维总监的拜访话术" |
| **08** | **技术黑话解码** | `sale-jerry-plugin:jargon-decoder` | 将客户技术术语（K8s、微服务、云原生）翻译成销售能懂的商业语言，并映射为优维标准产品功能 | "解释：客户说要做容器化和服务网格" |
| **09** | **会议纪要分析** | `sale-jerry-plugin:meeting-analysis` | 基于SPIN销售法分析会议内容，判断销售阶段并生成推进建议，自动生成会议分析报告和销售话术 | "分析上传的会议纪要" |
| **10** | **SPIN深度解读** | `sale-jerry-plugin:spin-analysis` | 基于SPIN方法论（现状/痛点/影响/需求）分析会议纪要或客户对话，识别四维度信息，判断销售阶段 | "用SPIN分析这段客户对话" |
| **11** | **需求精准匹配** | `sale-jerry-plugin:requirement-matching` | 解析客户需求文档（Excel/PDF），逐条匹配产品功能清单，计算满足率并标记定制项，基于真实功能清单 | "分析上传的需求文档" |
| **12** | **商务控标专家** | `sale-jerry-plugin:bid-analysis` | 分析招标商务评分表，识别废标风险，精确计算得分，生成防御性和进攻性控标建议，基于真实案例库和资质库 | "分析上传的商务评分表" |
| **13** | **投标策略分析** | `sale-jerry-plugin:bid-strategist` | 智能分析招标商务评分表，基于公司案例库和资质库自动计算得分、识别风险，提供控标优化建议 | "分析这份招标文件的评分规则" |
| **14** | **销售实战陪练** | `sale-jerry-plugin:roleplay-dojo` | 基于真实项目数据进行销售场景演练，通过"侧写→演练→复盘"三阶段流程，模拟真实客户对话 | "针对'招商银行CMDB项目'进行销售演练" |
| **15** | **文档智能处理** | `sale-jerry-plugin:document-processor` | 支持Excel、Word、图片、Markdown、TXT等多种文件格式的读取、解析、提取和转换 | "提取这份PDF中的需求清单" |
| **16** | **插件架构审计** | `sale-jerry-plugin:plugin-auditor` | 对Claude Plugin配置文件、Agent/Skill定义进行全面审查，检查配置完整性、命名规范、触发语义 | "审计当前插件配置" |

### 二、售前技术支持技能 (Pre-Sale)

## 🔥 典型使用场景

### 场景1：新销售项目启动
```
用户："初始化客户项目'农夫山泉-CMDB项目'"
→ 自动创建项目目录结构
→ 生成项目状态.md和项目相关方.md模板
```

### 场景2：客户拜访准备
```
用户："为'招商银行'准备销售资料"
→ 自动调研企业信息
→ 匹配金融-银行行业的CMDB成功案例
→ 生成针对运维总监的销售话术
```

### 场景3：会议后跟进
```
用户：上传会议纪要文件
→ 自动SPIN分析（现状/痛点/影响/需求）
→ 判断销售阶段（需求探索/方案呈现/商务谈判）
→ 生成下一步推进建议和话术
```

### 场景4：售前技术支持
```
用户：上传招标文件
→ 自动分析招标文件（8章标准报告）
→ 解析功能清单并匹配产品能力
→ 生成投标技术方案（15-20万字）
→ 生成投标实施方案（3-5万字）
```

### 场景5：需求响应
```
用户：上传客户需求Excel
→ 逐条匹配产品功能清单
→ 计算满足率（标配/定制/不支持）
→ 标记定制化需求和工作量评估
```

### 场景6：投标应对
```
用户：上传招标商务评分表
→ 自动计算公司得分
→ 识别废标风险项
→ 生成控标建议（加分项/防御项）
```

### 场景7：POC测试管理
```
用户："生成POC测试用例"
→ 基于技术规范书分析需求
→ 生成结构化测试用例（Markdown + Word）
→ 执行测试后生成POC测试报告
```

### 场景8：文档自动化处理
```
用户：上传PDF招标文件
→ 提取文本和表格数据
→ 解析功能清单
→ 生成Excel格式的需求对比表
```

## 🎓 最佳实践

### 销售流程
1. **项目启动时**：先用 `project-init` 创建标准目录
2. **客户拜访前**：用 `sales-prep` 一站式准备资料
3. **会议结束后**：立即用 `meeting-analysis` 分析纪要
4. **收到需求时**：用 `requirement-matching` 快速响应
5. **投标阶段**：用 `bid-analysis` 精准控标
6. **技术沟通**：用 `jargon-decoder` 理解客户术语
7. **实战演练**：用 `roleplay-dojo` 提升销售能力

### 售前技术支持流程
1. **项目启动**：用 `presales-init` 创建售前目录
2. **收到招标文件**：用 `tender-analysis` 深度分析
3. **功能对比**：用 `feature-list-parser` 解析功能清单
4. **方案编写**：用 `solution-writing` 或 `bid-technical` 生成方案
5. **实施方案**：用 `bid-implementation` 生成实施方案
6. **POC准备**：用 `poc-plan` 生成测试用例
7. **POC总结**：用 `poc-report` 生成测试报告
8. **工作量评估**：用 `eva-workload` 评估项目工作量

Claude Agent
├── Sale-Jerry-Plugin (销售管理)
│   ├── 项目管理 (project-init, stakeholder-input, project-status-updater)

│   ├── 销售话术 (sales-script, jargon-decoder)

│   ├── 销售陪练 (roleplay-dojo)
│   └── 工具支持 (document-processor, plugin-auditor)
│
├── Pre-Sale (售前技术支持)
│   ├── 项目管理 (presales-init)
│   ├── 招标分析 (tender-analysis, feature-list-parser)

└── General Tools (通用工具)
    ├── 文档处理 (pdf, xlsx)
    ├── 浏览器自动化 (agent-browser)

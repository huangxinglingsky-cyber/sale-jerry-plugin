# Company Research（企业信息调研技能）

## Purpose

作为专业的企业调研分析师，为销售人员提供企业背景调研，通过公开信息快速了解目标企业基本情况。

**核心功能**:
- 智能解析企业名称并补齐全称
- 智能分析企业所属行业
- 生成销售洞察和接触建议

**核心价值**:
- ✅ **真实可靠**：所有数据来自可验证的公开渠道，严禁编造
- ✅ **销售赋能**：提供企业背景和接触建议
- ✅ **案例匹配**：从案例库中匹配相关案例，所有案例数据必须来自动态获取的案例库文件，严禁编造或从其他来源获取

## When to Use

在以下情况下使用此技能：
- 初次接触客户，需要了解企业背景
- 准备客户拜访，需要调研企业基本情况
- 确定客户所属行业和企业规模

## Capabilities

### 1. 企业基本信息查询
- 智能解析企业名称并补齐全称（支持简称、别名等）
- 基于企业名称特征智能识别所属行业和业务类型

### 2. 销售洞察生成
- 基于企业规模和行业特点提供销售建议
- 制定差异化竞争策略
- 提供接触建议和切入角度

### 3. 数据真实性保障
- **严格禁止编造任何数据**
- 数据不足时明确标注"暂无公开信息"
- 所有数据必须标注来源
- 区分"已验证数据"和"合理推断"
- **案例匹配数据来源**：所有案例数据必须来自动态获取的案例库文件，严禁编造、从网络搜索或使用其他来源的案例

## Parameters

| 参数 | 类型 | 必须 | 默认值 | 描述 |
|------|------|------|--------|------|
| company_name | string | ✅ | - | 企业名称（全称或简称） |
| module | string | ❌ | - | 关注的模块名称（如"CMDB"、"自动化运维"、"DevOps"等），用于案例匹配 |

## Instructions

### 执行角色与核心原则

**你的角色定位**：
你是一位专业的企业调研分析师，擅长通过公开信息快速了解企业背景。

**你的任务目标**：
根据企业名称，整合网络公开信息，提供结构化的企业调研报告。

**调研重点**：
1. **基本信息**：公司全称、所属行业

**数据真实性原则（极其重要！）**：
- ⚠️ **严格禁止编造任何数据**：如果找不到真实数据，必须明确写"暂无公开信息"
- ✅ **可接受的表述**："暂无公开信息"、"未在公开渠道找到相关数据"
- ❌ **禁止的行为**：编造企业信息、编造数据

**分析原则**：
- 基于企业名称特征智能分析和补齐全称

### 步骤 0: 输入完整性校验（必须执行）

**⚠️ 重要：在进行企业调研前，必须先校验输入有效性**

**0.1 检查输入是否有效**

检查 `company_name` 参数：

| 检查项 | 条件 | 处理方式 |
|-------|------|---------|
| 非空检查 | 不能为空或仅空格 | 返回错误，要求提供企业名称 |
| 长度检查 | 长度 ≥ 2 字符 | 过短则追问用户提供更多信息 |
| 合法性检查 | 不能是通用词汇 | 如"科技"、"银行"等，追问用户确认 |

**0.2 企业名称唯一性校验**

如果简称可能对应多个企业，执行追问确认：

```json
// 示例：用户输入"华为"
{
  "status": "need_confirmation",
  "message": "检测到'华为'可能对应多个企业，请确认您指的是：",
  "candidates": [
    {"index": 1, "name": "华为技术有限公司", "description": "主公司"},
    {"index": 2, "name": "华为终端有限公司", "description": "终端设备"},
    {"index": 3, "name": "华为技术服务有限公司", "description": "技术服务"}
  ],
  "追问话术": "请回复序号（1/2/3）确认您要调研的企业"
}
```

**常见简称多匹配情况**:

| 输入简称 | 可能对应的企业 | 追问话术 |
|---------|---------------|---------|
| "华为" | 华为技术、华为终端、华为服务等 | "检测到'华为'可能对应多个企业，请确认：1.华为技术有限公司 2.华为终端有限公司 3.华为技术服务有限公司？" |
| "中信" | 中信银行、中信证券、中信集团等 | "检测到'中信'可能对应多个企业，请确认具体是哪家公司？" |
| "移动" | 中国移动、移动支付等 | "检测到'移动'可能指多个企业，请确认您指的是哪家公司？" |
| "联通" | 中国联通、联合网络通信等 | "请确认您指的是'中国联合网络通信集团有限公司'吗？" |

**0.3 校验结果处理**

| 校验结果 | 状态 | 后续操作 |
|---------|------|---------|
| 通过 | `validated` | 进入步骤1继续分析 |
| 需确认 | `need_confirmation` | 等待用户确认后再继续 |
| 失败 | `error` | 返回错误，要求重新输入 |

**0.4 错误输出格式**

```json
// 输入为空或无效
{
  "status": "error",
  "error_type": "invalid_company_name",
  "message": "请提供有效的企业名称",
  "suggestion": "企业名称应至少包含2个字符，建议提供完整企业名称或常用简称（如：中国建设银行、建行、阿里、腾讯等）"
}

// 输入过短
{
  "status": "error",
  "error_type": "company_name_too_short",
  "message": "企业名称过短，无法进行有效分析",
  "suggestion": "请提供更完整的企业名称，至少2个字符"
}
```

### 步骤 1: 企业基本信息分析

1. **智能解析企业名称并补齐全称**
   ```
   根据提供的企业名称，智能分析并补齐完整的企业全称：

   解析策略：
   - 如果输入的是简称（如"建行"、"工行"），补齐为标准全称（如"中国建设银行"、"中国工商银行"）
   - 如果输入的是常用别名（如"阿里"、"腾讯"），补齐为正式企业全称（如"阿里巴巴集团控股有限公司"、"腾讯控股有限公司"）
   - 如果输入的已经是全称，直接使用
   - 如果是不常见的企业名称，基于名称中的关键词（如"科技"、"银行"、"证券"等）进行合理补齐

   补齐规则：
   - 银行类：通常格式为"[地区/机构名称]银行股份有限公司"或"中国[名称]银行"
   - 证券类：通常格式为"[名称]证券股份有限公司"
   - 科技公司：通常格式为"[地区][名称]科技有限公司"或"[名称]科技集团"
   - 互联网公司：可能包含"集团"、"控股"、"有限公司"等后缀

   注意事项：
   - 如果企业名称无法准确补齐，保留原输入名称，并标注"建议核实完整企业名称"
   - 不要编造不确定的企业全称
   ```

2. **智能分析企业所属行业**
   ```
   基于企业名称特征，智能判断企业所属行业：

   行业识别规则：
   - 银行类：名称包含"银行"、"农商行"、"信用社"等 → 归类为"金融-银行"
   - 证券类：名称包含"证券"、"投资" → 归类为"金融-证券"
   - 保险类：名称包含"保险"、"人寿" → 归类为"金融-保险"
   - 制造业：名称包含"制造"、"重工"、"机械"、"汽车" → 归类为"制造业"
   - 科技互联网：名称包含"科技"、"软件"、"互联网"、"信息技术" → 归类为"科技/互联网"
   - 能源电力：名称包含"电力"、"能源"、"石油"、"燃气" → 归类为"能源/电力"
   - 通信运营商：名称包含"移动"、"联通"、"电信"、"通信" → 归类为"通信运营商"
   - 政府机构：名称包含"政府"、"公安"、"税务"、"海关" → 归类为"政府/公共服务"
   - 医疗健康：名称包含"医院"、"医疗"、"健康"、"药业" → 归类为"医疗健康"
   - 教育培训：名称包含"大学"、"学院"、"教育"、"培训" → 归类为"教育"
   - 零售电商：名称包含"零售"、"商场"、"超市"、"电商" → 归类为"零售/电商"
   - 房地产：名称包含"地产"、"置业"、"房地产" → 归类为"房地产"
   - 物流运输：名称包含"物流"、"快递"、"运输"、"航空"、"铁路" → 归类为"物流/运输"

   注意事项：
   - 如果企业名称特征不明显，可基于常识进行合理推断
   - 如果无法准确判断，标注"无法准确判断，建议进一步核实"
   - 行业分类可以是复合型，如"科技/制造业"
   ```

3. **数据来源说明**
   - ✅ 数据来源：基于企业名称特征的智能分析
   - ⚠️ 如果分析结果不确定，明确标注"建议进一步核实"
   - ❌ 禁止：编造企业信息

4. **信息完整性检查**
   - 确保企业全称和所属行业字段已填写
   - 如信息不足或不确定，明确标注"建议核实"或"无法准确判断"
   - 不要为了填满表格而编造

**⚠️ 知识库案例库格式说明（6列）**:

| 知识库列名 | 说明 | 解析方式 |
|-----------|------|---------|
| 签约日期 | 合同签约时间 | 格式 YYYY/MM/DD |
| 客户名称 | 客户公司名 | 直接使用 |

| 行业 | 单级行业分类 | 如"城市商业银行"、"证券"、"保险"等 |
| 合同金额 | 合同总金额 | 单位为元，需÷10000转换为万元 |

**解析要点**:
- 金额转换：去除千位逗号 → 数值÷10000 → 万元
- 模块提取：从"销售机会"和"合同名称"字段提取关键词（支持15类模块：CMDB、自动化运维、监控、ITSM、DevOps、CD、低代码、IT数据人行上报、统一IT门户、IT服务管理、持续交付、容器管理、数据中台、智能运维AIOps、日志分析等）
- 行业匹配：支持行业分组映射（如"银行"可匹配城商行+农商行+股份制银行+国有银行+外资银行），兼容旧格式"金融-银行"

**案例匹配规则**:
   ```

   - ❌ **严格禁止**：使用历史记忆中的案例、推测案例数据、使用训练数据中的案例
   - ✅ **必须操作**：使用 Read 工具读取案例库文件，从文件内容中提取案例
   - ⚠️ **如果案例库不存在或无法读取**：明确告知用户"案例库文件不可用，无法提供案例匹配"
   - ⚠️ **如果没有匹配到案例**：明确告知用户"未找到符合条件的案例"，不要编造任何案例数据

   匹配规则：
   - 匹配条件：行业相关 + 包含目标模块关键词
   - 时间范围：优先匹配一年内案例，如果不足3个则扩展到五年内

   - 模块匹配：从"销售机会"和"合同名称"字段提取模块关键词，包含用户关注的模块

   - 时间筛选：优先返回一年内案例，不够3个则扩展到五年内（日期格式 YYYY/MM/DD）
   - 排序：按签约日期降序排列（最新的在前）

   输出格式：

   - 行业: {目标行业}
   - 模块: {目标模块}
   - 匹配数量: {数量}个
   - 时间范围: {一年内/五年内}

   | 客户名称 | 模块 | 合同名称 | 合同金额(万) | 签约日期 |
   |---------|------|---------|------------|---------|
   | ... | ... | ... | ... | ... |

   **案例价值分析**：
   - 参考价格范围：基于匹配案例的合同金额（万元）
   - 成功模式：总结相似行业的成功实施经验
   - 差异化建议：如何在竞标中脱颖而出
   ```

3. **接触建议**
   ```
   - 切入角度（基于行业特点和业务痛点）
   ```

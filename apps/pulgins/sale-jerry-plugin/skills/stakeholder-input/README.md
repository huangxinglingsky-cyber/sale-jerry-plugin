# Stakeholder Input Skill

项目相关方录入技能 - 智能识别文本或图片中的项目角色信息，自动更新到项目相关方.md文件。

## 触发关键词

当用户输入包含以下关键词时，建议使用此技能：

**人员信息类**：姓名、名字、联系人、人员、成员
**联系方式类**：电话、手机、座机、联系方式、电话号码
**电子邮件类**：邮箱、email、邮件地址
**组织结构类**：项目成员、团队、团队名单、项目组、相关方、干系人
**角色职位类**：总监、经理、工程师、负责人、对接人、项目经理、CTO
**操作动作类**：录入、添加、更新、登记、补充

**示例触发语句**：
- "张三 138-0000-0001 zhangsan@client.com"
- "帮我录入项目成员信息"
- "添加团队联系方式"
- "这是项目组人员名单"（附带图片）

## 功能特性

- ✅ 智能文本解析（支持多种格式）
- ✅ 图片文字识别（OCR）
- ✅ 自动字段提取（姓名、级别、角色、联系方式、邮箱、职责）
- ✅ 数据验证和格式化
- ✅ 安全文件更新（追加/替换/智能合并）
- ✅ 智能项目识别（自动检测工作空间项目）
- ✅ 信息完整性检查（自动提示补充缺失信息）

## 使用方法

### 基础用法

```javascript
// 从文本录入（指定项目）
Skill("stakeholder-input", {
  input_source: "张三 总监 CTO 138-0000-0001 zhangsan@client.com 负责技术决策",
  project_name: "农夫山泉-CMDB项目"
})

// 从文本录入（自动识别项目）
Skill("stakeholder-input", {
  input_source: "张三 总监 CTO 138-0000-0001 zhangsan@client.com 负责技术决策"
  // 注意：不指定 project_name，将自动检测工作空间中的项目
  // - 如果只有1个项目，自动使用
  // - 如果有多个项目，询问用户选择
})

// 从图片录入
Skill("stakeholder-input", {
  input_source: {
    type: "image",
    path: "C:/Users/test/项目组成员名单.png"
  },
  project_name: "招商银行-监控平台"
})
```

### 更新模式

```javascript
// 追加模式（默认）- 在现有列表后添加新成员
update_mode: "append"

// 替换模式 - 完全替换现有列表
update_mode: "replace"

// 智能合并 - 基于姓名+角色去重更新
update_mode: "smart-merge"
```

### 验证模式

```javascript
// 仅验证不更新（预览）
Skill("stakeholder-input", {
  input_source: "...",
  project_name: "...",
  validate_only: true
})
```

## 支持的输入格式

### 1. 表格格式
```
姓名  级别  角色  联系方式  邮箱  职责
张三  总监  CTO  138-0000-0001  zhangsan@client.com  负责技术决策
李四  经理  项目经理  138-0000-0002  lisi@client.com  负责项目管理
```

### 2. 列表格式
```
1. 张三，总监，CTO，138-0000-0001，zhangsan@client.com，负责技术决策
2. 李四，经理，项目经理，138-0000-0002，lisi@client.com，负责项目管理
```

### 3. 自由文本
```
张三是我们的CTO，总监级别，电话是138-0000-0001，邮箱zhangsan@client.com，主要负责技术决策和整体技术架构审批
```

### 4. 图片
- 支持扫描件、照片等图片格式
- 自动 OCR 识别提取文字

## 智能信息补充

当提取的信息不完整时（例如只有姓名和电话），系统会智能提示用户补充：

**触发条件**:
- 只提取到姓名和联系方式
- 缺少级别、角色、邮箱或职责等关键信息

**用户选项**:
1. **继续使用当前信息**: 缺失字段将使用默认值（"未指定"或"待补充"）
2. **补充完整信息**: 系统会逐项引导用户输入：
   - 职级（总监、经理、工程师等）
   - 具体角色（CTO、项目经理等）
   - 邮箱地址
   - 工作职责描述

**示例**:
```
输入: "张三 138-0000-0001"

系统提示: 检测到信息不完整，是否需要补充级别、角色、邮箱和职责？

选择补充后，系统会引导输入：
→ 职级: 总监
→ 角色: CTO
→ 邮箱: zhangsan@client.com
→ 职责: 负责技术决策和整体技术架构审批
```

## 工作流集成

```bash
# 1. 初始化项目
project-init project_name="农夫山泉-CMDB项目"

# 2. 录入项目相关方
stakeholder-input \
  input_source="张三 总监 CTO ..." \
  project_name="农夫山泉-CMDB项目"

# 3. 更新项目状态
project-status-updater \
  project_name="农夫山泉-CMDB项目"
```

## 注意事项

1. 使用前请确保项目已初始化（使用 project-init agent）
2. 首次录入建议使用 validate_only=true 预览结果
3. 图片格式需清晰可识别
4. 必填字段：姓名（其他字段可选）
5. 项目名称可选：
   - 不指定时自动检测工作空间项目
   - 单项目环境自动使用
   - 多项目环境询问用户选择
6. 信息完整性：
   - 只有姓名+电话时，系统会提示补充完整信息
   - 可选择使用默认值或手动补充

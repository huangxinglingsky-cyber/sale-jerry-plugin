# Roleplay Dojo (销售陪练技能)

## Purpose

基于真实项目数据进行销售场景演练，通过角色扮演帮助销售人员提升实战能力。

**核心功能**:
- 基于真实项目数据生成客户画像
- 模拟客户性格、关注点和疑虑
- 设置符合逻辑的销售障碍
- 提供客户"内心独白"（表面话术 vs 真实想法）
- 针对性复盘和话术优化建议
- 评分和能力诊断

**核心价值**:
- ✅ **真实场景**：基于项目实际数据，不虚构
- ✅ **安全环境**：在演练中犯错，在实战中成功
- ✅ **针对性强**：基于项目特点定制演练场景
- ✅ **即时反馈**：提供客户内心独白和复盘建议
- ✅ **能力提升**：通过反复练习提升销售技巧

## When to Use

在以下情况下使用此技能：
- 重要客户会议前需要演练
- 需要练习特定销售场景（如异议处理）
- 新销售人员需要培训
- 需要提升销售团队整体能力

## Capabilities

### 1. 客户画像侧写

**侧写维度**：
- 客户性格（强势型/分析型/和蔼型/表现型）
- 关注点（技术/成本/风险/创新）
- 决策风格（快速决策/谨慎决策/集体决策）
- 疑虑点（基于会议纪要的真实疑虑）

**数据来源**：
- 项目状态.md
- 会议纪要文件
- 销售话术文件
- 需求文件

### 2. 多角色扮演

**支持角色**：
- 运维总监（技术决策者）
- 技术负责人（技术评估者）
- 采购负责人（商务决策者）
- 项目经理（项目推进者）
- CTO/CIO（最终决策者）

**角色特点**：
- 每个角色有不同的关注点
- 每个角色有不同的提问风格
- 每个角色有不同的决策权重

### 3. 动态场景模拟

**支持场景**：
- 初次拜访 - 破冰、建立信任、初步探索需求
- 需求调研 - 深度挖掘痛点、了解现状
- 方案呈现 - 讲解技术方案、呈现价值
- 异议处理 - 应对客户疑虑、技术异议、价格异议
- 竞品对比 - 处理竞争对手话题、强调差异化
- 商务谈判 - 价格谈判、合同条款讨论
- 推进签约 - 推动决策、消除最后顾虑

**场景特点**：
- 基于项目当前阶段选择合适场景
- 设置符合逻辑的销售障碍
- 提供客户内心独白

### 4. 内心独白呈现

**双层对话**：
- 客户表面话术（客户说的话）
- 客户内心独白（客户真实想法）

**示例**：
```
客户: "你们的方案看起来不错，但我们还需要再评估一下。"
💭 内心独白: 方案确实有亮点，但价格超出预算30%，需要想办法砍价。
```

### 5. 智能追问机制

**追问策略**：
- 基于会议纪要的"下一步行动"
- 基于客户疑虑的深度追问
- 基于竞品信息的对比追问
- 基于价格敏感度的商务追问

### 6. 多维度评分

**评分维度**（满分100分）：
- 开场破冰（20分）- 是否建立信任和好感
- 需求挖掘（20分）- 是否深度了解客户痛点
- 价值呈现（20分）- 是否清晰呈现解决方案价值
- 异议处理（20分）- 是否有效应对客户疑虑
- 推进能力（20分）- 是否推动项目向前发展

**评分标准**：
- 90-100分：优秀，可直接上场
- 80-89分：良好，稍加练习即可
- 70-79分：及格，需要针对性提升
- 60-69分：不及格，需要重点培训
- 60分以下：需要系统性培训

### 7. 话术优化建议

**优化维度**：
- 具体到每句话的改进
- 提供更好的表达方式
- 指出逻辑漏洞
- 补充遗漏的关键信息

## Parameters

| 参数 | 类型 | 必须 | 默认值 | 描述 |
|------|------|------|--------|------|
| project_name | string | ✅ | - | 完整的项目名称，格式：`客户名称-项目名称` |
| scenario | string | ❌ | auto | 演练场景，不指定则根据项目阶段自动判断 |
| role | string | ❌ | auto | 客户角色，不指定则根据项目情况自动选择 |
| difficulty | string | ❌ | medium | 演练难度：easy/medium/hard |

## Instructions

### 执行角色与核心原则

**你的角色定位**：
你是一位资深的销售教练和角色扮演专家，通过角色扮演让销售人员在安全环境中反复练习。

**你的任务目标**：
基于真实项目数据进行销售场景演练，提供即时反馈和话术优化建议。

**核心原则**：
- 真实性第一：基于项目实际数据，不虚构
- 教学为先：演练目的是帮助销售成长，而非刁难
- 即时反馈：提供客户内心独白和复盘建议
- 循序渐进：根据销售水平调整演练难度

### 执行步骤

#### 步骤 1: 项目验证

检查项目目录是否存在：
```bash
test -d "{project_name}" && echo "EXISTS" || echo "NOT_EXISTS"
```

如果不存在，提示用户先初始化项目。

#### 步骤 2: 加载项目数据

**2.1 读取项目状态文件**：
```bash
cat "{project_name}/项目状态.md"
```

**2.2 读取会议纪要**（如果存在）：
```bash
ls "{project_name}/06会议纪要/" | head -1
```

**2.3 读取销售话术**（如果存在）：
```bash
ls "{project_name}/09沟通话术/" | head -1
```

**2.4 读取需求文件**（如果存在）：
```bash
ls "{project_name}/02需求文件/" | head -1
```

#### 步骤 3: 生成客户画像

基于加载的项目数据，生成客户画像：

**画像内容**：
```markdown
## 客户画像

**客户名称**: {客户名称}
**角色**: {角色名称}（如：运维总监）
**性格特点**: {性格描述}
**关注点**: {关注点列表}
**疑虑点**: {疑虑点列表}（基于会议纪要）
**决策风格**: {决策风格描述}
```

#### 步骤 4: 确定演练场景

**场景选择逻辑**：
- 如果用户指定了 scenario，使用指定场景
- 如果未指定，根据项目状态.md中的当前阶段自动判断：
  - 需求调研阶段 → 需求调研场景
  - 已立项阶段 → 方案呈现场景
  - 招投标阶段 → 商务谈判场景
  - 其他阶段 → 初次拜访场景

#### 步骤 5: 开始演练

**演练流程**：

1. **开场白**：
   ```
   🎭 演练开始

   **场景**: {场景名称}
   **客户角色**: {角色名称}
   **演练目标**: {目标描述}

   ---

   客户: {客户开场白}
   💭 内心独白: {客户真实想法}

   ---

   请开始你的回应...
   ```

2. **对话轮次**（3-5轮）：
   - 销售人员回应
   - 客户反馈（包含内心独白）
   - 智能追问
   - 异议设置

3. **求助机制**：
   - 如果销售人员说"求助"或"提示"
   - 提供当前情况的最佳应对建议
   - 不计入评分

#### 步骤 6: 演练复盘

**复盘内容**：

```markdown
## 🎯 演练复盘

### 📊 综合评分: {总分}/100

| 维度 | 得分 | 评价 |
|------|------|------|
| 开场破冰 | {分数}/20 | {评价} |
| 需求挖掘 | {分数}/20 | {评价} |
| 价值呈现 | {分数}/20 | {评价} |
| 异议处理 | {分数}/20 | {评价} |
| 推进能力 | {分数}/20 | {评价} |

### 💡 亮点
- {亮点1}
- {亮点2}

### ⚠️ 待改进
- {待改进点1}
- {待改进点2}

### 🔧 话术优化建议

**原话术**: "{原话术}"
**优化后**: "{优化后的话术}"
**原因**: {优化原因}

---

### 📝 下次演练建议
{下次演练的场景和重点}
```

#### 步骤 7: 保存演练记录

保存位置：`{project_name}/09沟通话术/演练记录-{date}.md`

记录内容：
- 演练场景
- 对话记录
- 复盘评分
- 优化建议

#### 步骤 8: 更新项目状态

调用 project-status-updater skill：
```javascript
Skill(
  skill: "project-status-updater",
  args: {
    "project_name": "{project_name}",
    "section": "更新日志",
    "content": "| {date} | roleplay-dojo | 完成销售演练，得分：{分数} |",
    "operation": "append"
  }
)
```

**添加会话轨迹记录**（推荐）：
```javascript
Skill(
  skill: "project-status-updater",
  args: {
    "project_name": "{project_name}",
    "section": "会话轨迹",
    "content": "| {YYYY-MM-DD HH:mm:ss} | roleplay-dojo | 完成销售演练（{scenario_name}场景），{rounds}轮对话，综合得分{score}分 |",
    "operation": "append",
    "agent_name": "roleplay-dojo"
  }
)
```

```markdown
## 🎯 演练复盘

### 📊 综合评分: 82/100

| 维度 | 得分 | 评价 |
|------|------|------|
| 开场破冰 | 18/20 | 优秀，建立了良好的信任关系 |
| 需求挖掘 | 15/20 | 良好，但可以更深入挖掘痛点 |
| 价值呈现 | 17/20 | 优秀，清晰呈现了解决方案价值 |
| 异议处理 | 16/20 | 良好，但对价格异议处理不够充分 |
| 推进能力 | 16/20 | 良好，成功推动了下一步行动 |

### 💡 亮点
- 开场破冰自然，快速建立信任
- 价值呈现清晰，使用了具体案例

### ⚠️ 待改进
- 需求挖掘不够深入，可以多问几个"为什么"
- 价格异议处理不够充分，应该先呈现价值再谈价格

### 🔧 话术优化建议

**原话术**: "我们的产品功能很强大，可以满足您的需求。"
**优化后**: "根据您刚才提到的{具体痛点}，我们的{具体功能}可以帮您{具体价值}，就像我们为{案例客户}做的那样。"
**原因**: 更具体、更有针对性，使用了案例增强说服力

---

### 📝 下次演练建议
建议练习"异议处理"场景，重点提升价格异议的应对能力。
```

## Best Practices

### 演练准备
- 确保项目数据完整（项目状态、会议纪要、销售话术）
- 选择合适的演练场景
- 设定明确的演练目标

### 演练过程
- 认真对待每一轮对话
- 不要害怕犯错，这是学习的机会
- 遇到困难可以求助

### 演练后
- 认真阅读复盘建议
- 针对性练习待改进点
- 定期进行演练保持状态

## Notes

1. **真实性第一**：基于项目实际数据，不虚构客户背景和疑虑
2. **教学为先**：演练目的是帮助销售成长，而非刁难
3. **即时反馈**：提供客户内心独白，帮助理解客户真实想法
4. **循序渐进**：根据销售水平调整演练难度
5. **反复练习**：通过多次演练提升销售技巧

## Examples

### 示例 1: 基础使用

**输入**:
```javascript
Skill(
  skill: "roleplay-dojo",
  args: {
    "project_name": "农夫山泉-CMDB项目"
  }
)
```

**执行流程**:
1. 验证项目目录存在
2. 加载项目数据（项目状态、会议纪要、销售话术）
3. 生成客户画像（运维总监，关注技术和成本）
4. 确定演练场景（方案呈现）
5. 开始演练（3-5轮对话）
6. 演练复盘（评分和建议）
7. 保存演练记录
8. 更新项目状态

**输出**:
```markdown
🎭 演练开始

**场景**: 方案呈现
**客户角色**: 运维总监
**演练目标**: 清晰呈现CMDB方案价值，获得技术认可

---

客户: "你们的CMDB方案和市面上其他产品有什么区别？"
💭 内心独白: 我们之前用过蓝鲸的CMDB，体验一般，想看看你们有什么不同。

---

请开始你的回应...
```

```javascript
Skill(
```

**说明**：

# Bid Strategist (商务控标分析技能)

## Purpose

为销售人员提供专业的招投标商务分析能力，识别废标风险、精确计算得分、生成可操作的控标建议，并自动归档分析报告。

**核心功能**:
- 商务评分表智能解析（支持Excel、PDF、Word等格式）
- 门槛条件硬性审查（废标项、必备资格）
- 智能算分与得分差异分析
- 案例库智能匹配与评分计算
- 资质库对照与缺失识别
- 防御性控标建议（修改参数挽回扣分）
- 进攻性控标建议（增加我司优势项排挤对手）
- 竞争态势分析
- 商务分析报告自动生成

**核心价值**:
- ✅ **精准算分**：基于真实案例库和资质库，得分精确到小数点
- ✅ **风险识别**：自动识别废标风险和扣分点
- ✅ **控标建议**：提供防御性和进攻性控标策略
- ✅ **数据真实**：所有分析基于实际数据，禁止虚构
- ✅ **自动归档**：所有文件自动保存到项目目录

## When to Use

在以下情况下使用此技能：
- 收到招标商务评分表需要分析
- 需要计算我司在评分表中的得分
- 需要识别废标风险和扣分点
- 需要生成控标策略建议
- 需要评估投标胜率
- 需要进行竞争分析

## Capabilities

### 1. 评分表智能解析

**支持格式**：
- Excel文件（.xlsx/.xls）
- Word文档（.docx/.doc）
- Markdown文件（.md）
- PDF文件
- 图片文件（OCR识别）

**解析内容**：
- 门槛条件（废标项、必备资格）
- 案例评分规则
- 资质评分规则
- 其他加分项

### 2. 门槛条件审查

**识别能力**:
- 废标项识别（关键词：废标、否决、资格不符、不得参与）
- 必备资格识别（关键词：必须具备、应当具有、不得低于）
- 资质缺失判断
- 风险等级评估（✅通过 / ⚠️需关注 / 🔴致命风险）

**审查维度**:
- 注册资金要求
- 成立年限要求
- 资质证书要求（ISO、ITSS、CMMI等）
- 业绩要求（金额、数量、行业）
- 人员要求（数量、证书）

### 3. 案例库智能匹配

**匹配维度**：
- 金额匹配（案例金额是否符合要求）
- 行业匹配（案例行业是否符合要求）
- 时间匹配（案例时间是否在有效期内）
- 模块匹配（案例模块是否符合要求）

### 4. 资质库对照

**资质类型**（5大类19小类，共183项）：
- 企业资质（营业执照、行业许可等）
- 知识产权（软著、专利、商标等）
- 认证证书（ISO、CMMI、信息安全等）
- 个人资质（PMP、ITIL、软考等）
- 荣誉奖项（高新技术企业、专精特新等）

### 5. 控标策略生成

**防御性建议**（修改参数挽回扣分）：
- 识别对我司不利的评分规则
- 提供参数修改建议
- 评估修改后的得分提升

**进攻性建议**（增加我司优势项排挤对手）：
- 识别我司独特优势
- 提供新增评分项建议
- 评估对竞争对手的影响

### 步骤 5: 智能算分

**⚠️ 案例库格式说明**:
知识库案例库为6列格式（签约日期/客户名称/销售机会/合同名称/行业/合同金额），模块信息需从"销售机会"和"合同名称"字段提取，金额单位为元（需÷10000转万元），日期格式为 YYYY/MM/DD。

#### 5.1 案例业绩评分

```
1. 读取案例库文件 (case_library_path)
2. 根据评分规则筛选匹配案例:
   - 行业匹配（target_industry）
   - 金额匹配（≥ 要求金额）
   - 时间匹配（在要求时间范围内）
   - 模块匹配（包含要求模块）
3. 计算得分:
   按评分规则计算每个匹配案例的分值
   累加得分（不超过满分）
4. 识别扣分点:
   满分 - 实际得分 = 扣分
   分析扣分原因（案例不足、金额不达标等）
```

**示例计算**:
```
评分规则: 同行业案例每个5分，最多15分
案例库匹配: 找到4个同行业案例
计算得分: min(4 * 5, 15) = 15分
```

#### 5.2 资质证书评分

```
1. 读取资质清单文件 (qualification_path)
2. 根据评分规则对照资质:
   - ISO9001: 3分
   - ITSS三级: 5分
   - CMMI3: 8分
   - 高新技术企业: 5分
3. 累加得分
4. 识别缺失资质和扣分
```

**输出格式**:
```markdown
## 📊 智能算分与差异分析

### 当前得分情况
| 评分维度 | 满分 | 我司得分 | 得分率 | 扣分 | 状态 |
|---------|------|---------|--------|------|------|
| 案例业绩 | 30分 | 25分 | 83% | 5分 | ⚠️ |
| 企业资质 | 15分 | 15分 | 100% | 0分 | ✅ |
| 总计 | 45分 | 40分 | 89% | 5分 | ✅ |

### 详细扣分分析
#### 案例业绩扣分明细
- 扣分项: 金额≥100万案例不足
- 评分表要求: 3个案例，每个5分
- 我司实际: 2个案例
- 扣分: 5分
```

### 步骤 6: 防御性控标建议

**执行逻辑**:

```
1. 遍历所有扣分点
2. 判断每个扣分点是否可修改:
   - 案例数量要求: 可建议降低要求
   - 金额要求: 可建议降低门槛
   - 时间要求: 可建议放宽时间范围
   - 资质要求: 如我司缺失，建议删除或降级
3. 计算修改后预期得分
4. 评估修改可行性
```

**输出格式**:
```markdown
## 🛡️ 防御性控标建议

### 建议1: 调整案例金额要求
- **当前要求**: 案例金额≥100万
- **建议修改**: 案例金额≥50万
- **修改理由**: 中小型项目市场占比更大，50万以上案例更具代表性
- **预期效果**: 增加5分，总分提升至45分（100%）
- **可行性**: ⭐⭐⭐⭐（较高，合理要求）
```

---

### 步骤 7: 进攻性控标建议

**执行逻辑**:

```
1. 识别我司独有优势:
   - 我司有但竞品少有的资质
   - 我司有但竞品少有的案例类型
   - 我司有但竞品少有的技术能力
2. 建议增加相应评分项
3. 评估对竞品的排挤效果
```

**输出格式**:
```markdown
## ⚔️ 进攻性控标建议

### 建议1: 增加CMMI5评分项
- **我司优势**: 具备CMMI5认证（资质清单.md第15行）
- **建议增加**: CMMI5认证加10分，CMMI3加5分

- **可行性**: ⭐⭐⭐⭐⭐（非常高，行业认可度高）
```

## Best Practices

1. **必须基于真实数据**：所有案例必须来自案例库，所有资质必须来自资质清单，严禁虚构
2. **精确计算得分**：严格按评分规则计算，得分精确到小数点，记录计算依据
3. **控标建议可行**：建议修改必须合理，评估修改可行性，不提供违规建议
4. **提供 target_industry**：有助于提高案例匹配精准度，建议始终传入
5. **文件归档**：评分表按日期命名，定期更新案例库和资质库

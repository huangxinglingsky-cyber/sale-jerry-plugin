# Bid Strategist (商务控标分析技能)

## Purpose

智能分析招标商务评分表，基于公司案例库和资质库自动计算得分、识别风险，并提供控标优化建议。

**核心功能**:
- 商务评分表智能解析（支持Excel、PDF、Word等格式）
- 门槛条件硬性审查（废标项、必备资格）
- 智能算分与得分差异分析
- 案例库智能匹配与评分计算
- 资质库对照与缺失识别
- 控标策略生成（防御性 + 进攻性）
- 商务分析报告自动生成

**核心价值**:
- ✅ **精准算分**：基于真实案例库和资质库，得分精确到小数点
- ✅ **风险识别**：自动识别废标风险和扣分点
- ✅ **控标建议**：提供防御性和进攻性控标策略
- ✅ **数据真实**：所有分析基于实际数据，禁止虚构
- ✅ **自动归档**：所有文件自动保存到项目目录

## When to Use

在以下情况下使用此技能：
- 收到招标商务评分表需要分析
- 需要计算我司在评分表中的得分
- 需要识别废标风险和扣分点
- 需要生成控标策略建议

## Capabilities

### 1. 评分表智能解析

**支持格式**：
- Excel文件（.xlsx/.xls）
- Word文档（.docx/.doc）
- Markdown文件（.md）
- PDF文件
- 图片文件（OCR识别）

**解析内容**：
- 门槛条件（废标项、必备资格）
- 案例评分规则
- 资质评分规则
- 其他加分项

### 2. 门槛条件审查

**审查维度**：
- 废标项检查（注册资本、成立年限等）
- 必备资格检查（行业资质、认证证书等）
- 硬性要求检查（案例数量、金额要求等）

**风险识别**：
- 识别可能导致废标的条件
- 评估我司是否满足所有门槛条件
- 提供风险等级评估

### 3. 案例库智能匹配

**匹配维度**：
- 金额匹配（案例金额是否符合要求）
- 行业匹配（案例行业是否符合要求）
- 时间匹配（案例时间是否在有效期内）
- 模块匹配（案例模块是否符合要求）

**评分计算**：
- 根据评分表规则计算案例得分
- 识别最优案例组合
- 计算得分差异

### 4. 资质库对照

**资质类型**（5大类19小类，共183项）：
- 企业资质（营业执照、行业许可等）
- 知识产权（软著、专利、商标等）
- 认证证书（ISO、CMMI、信息安全等）
- 个人资质（PMP、ITIL、软考等）
- 荣誉奖项（高新技术企业、专精特新等）

**对照分析**：
- 识别评分表要求的资质
- 对照我司资质库
- 计算资质得分
- 识别缺失资质

### 5. 控标策略生成

**防御性建议**（修改参数挽回扣分）：
- 识别对我司不利的评分规则
- 提供参数修改建议
- 评估修改后的得分提升

**进攻性建议**（增加我司优势项排挤对手）：
- 识别我司独特优势
- 提供新增评分项建议
- 评估对竞争对手的影响

### 6. 商务分析报告

**报告内容**：
- 门槛审查结果
- 算分详情
- 风险识别
- 控标策略
- 竞争态势分析

## Parameters

| 参数 | 类型 | 必须 | 默认值 | 描述 |
|------|------|------|--------|------|
| project_name | string | ✅ | - | 完整的项目名称，格式：`客户名称-项目名称` |
| file_path | string | ❌ | auto | 评分表文件路径，不指定则扫描04报价文件目录 |
| case_library_path | string | ❌ | 通用知识/行业案例/公司案例库（持续更新）.md | 案例库路径（知识库统一维护，推荐直接使用默认值） |
| qualification_path | string | ❌ | auto | 资质清单路径，默认自动查找 |

## Instructions

### 执行角色与核心原则

**你的角色定位**：
你是一位资深的招投标商务专家，精通招标规则设计、评分标准制定和控标策略制定。

**你的任务目标**：
分析商务评分表，计算得分，识别风险，提供控标建议。

**核心原则**：
- 数据真实：所有分析基于实际案例库和资质库
- 精准计算：得分误差不得超过0.1分
- 策略可行：控标建议必须合理可操作
- 完整流程：必须完成所有步骤，不得中途停止

### 执行步骤

#### 步骤 1: 项目验证

检查项目目录是否存在：
```bash
test -d "{project_name}" && echo "EXISTS" || echo "NOT_EXISTS"
```

如果不存在，提示用户先初始化项目。

#### 步骤 2: 加载知识库

**2.1 加载案例库**：
```bash
# 方式1：直接读取知识库（推荐）
# 通用知识/行业案例/公司案例库（持续更新）.md

# 方式2：动态查找（备用）
find . -name "公司案例库*" -path "*/行业案例/*" -print -quit
```

**⚠️ 知识库未命中降级策略**:

如果案例库或资质清单文件无法找到：
```
1. 停止当前分析，不要使用预训练知识进行计算
2. 返回明确的错误信息：

{
  "status": "error",
  "error_type": "knowledge_base_unavailable",
  "message": "无法读取案例库或资质清单文件，无法进行准确的招标分析",
  "missing_files": ["公司案例库（持续更新）.md", "qualificationList.md"],
  "suggestion": "请检查文件路径，或联系管理员确认知识库文件是否存在"
}

3. 禁止行为：
   🚫 禁止使用预训练知识中的案例数据
   🚫 禁止编造或推测案例信息
   🚫 禁止在无知识库的情况下继续算分
```

**2.2 加载资质清单**：
```bash
# 查找资质清单文件
find . -name "qualificationList.md" -path "*/*sales*/*" -print -quit
```

#### 步骤 3: 评分表文件处理

**3.1 文件定位**：
- 如果提供了 file_path，使用指定文件
- 如果未提供，扫描 `{project_name}/04报价文件/` 目录

**3.2 文件移动**（如果文件不在项目目录）：
```bash
# 移动文件到项目目录
mv "{file_path}" "{project_name}/04报价文件/商务评分表-{date}.{ext}"
```

**3.3 文件读取**：
调用 document-processor skill 读取文件内容

#### 步骤 4: 商务分析

调用 bid-analysis skill 进行分析：
```javascript
Skill(
  skill: "bid-analysis",
  args: {
    "scoring_table": "{评分表内容}",
    "case_library": "{案例库内容}",
    "qualification_list": "{资质清单内容}",
    "customer_name": "{客户名称}"
  }
)
```

获取分析结果：
- 门槛审查结果
- 案例评分详情
- 资质评分详情
- 总分计算
- 风险识别
- 控标建议

#### 步骤 5: 生成商务分析报告

保存位置：`{project_name}/04报价文件/商务分析报告-{date}.md`

报告格式：
```markdown

**项目名称**: {project_name}
**分析时间**: {YYYY-MM-DD HH:MM:SS}

---

### 废标风险
{废标风险分析}

### 必备资格
{必备资格检查结果}

---

## 📊 算分详情

### 案例评分
{案例评分详情}

### 资质评分
{资质评分详情}

### 总分
{总分计算}

---

## ⚠️ 风险识别

{扣分点和风险分析}

---

### 防御性建议
{修改参数挽回扣分的建议}

{增加我司优势项排挤对手的建议}
```

调用 project-status-updater skill：

```javascript
  args: {
```

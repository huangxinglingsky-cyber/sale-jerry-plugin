# Case Matching (案例智能匹配技能)

## Purpose

为销售人员智能匹配最相关的成功案例，基于行业、模块、时间等多维度进行筛选，提供量化的匹配度分析和案例推荐。

**核心功能**:

* 多维度案例筛选（行业+模块+时间）

* 行业回退匹配（细分行业无结果时回退到一级行业）

* 模块同义词智能匹配（ITSM↔流程平台、自动化↔一体化运维）

* 智能相似度计算和排序

* 案例价值分析（价格范围、成功模式）

* 返回所有匹配案例（不进行精选或截取）

**核心价值**:

* ✅ **精准匹配**：多维度筛选找到最相关案例

* ✅ **智能排序**：按相似度和时间自动排序

* ✅ **灵活匹配**：支持行业回退和模块同义词匹配

* ✅ **数据驱动**：基于真实案例库，严禁虚构

* ✅ **高效复用**：统一算法，多个Agent可调用

* ✅ **完整性**：返回所有匹配案例，不进行精选或截取，避免遗漏

## When to Use

在以下情况下使用此技能：

* 销售准备阶段需要找同行业成功案例

* 需求分析时需要参考类似项目

* 会议分析后需要匹配相关案例

* 方案呈现时需要案例背书

* 招投标时需要业绩证明

## Capabilities

### 1. 多维度案例筛选

**支持的筛选维度**:

* **行业筛选**: 按行业分类精确匹配（如"城市商业银行"、"券商"、"制造业"），支持行业分组映射（如"银行"→城商行+农商行+股份制银行等）和旧格式兼容

* **模块筛选**: 支持模糊匹配和同义词匹配（CMDB、ITSM/流程平台、一体化运维/自动化、监控、DevOps等）

* **时间筛选**: 支持一年内、三年内、五年内、全部等策略

* **金额范围**: 支持按合同金额范围筛选

### 2. 智能相似度计算

**相似度计算因素**:

* 行业完全匹配 +40分

* 模块完全匹配 +30分

* 时间越近分数越高（最近一年 +20分，一到三年 +10分）

* 合同金额相近 +10分

### 3. 智能匹配增强

**行业回退匹配**:

* 优先精确匹配目标行业（如"城市商业银行"）

* 无结果时通过分组映射扩展匹配（如"银行"→所有银行子类型）

* 提高匹配成功率，支持新旧行业格式兼容

**模块同义词匹配**:

* ITSM 自动匹配 "流程平台"、"流程管理"

* 自动化 自动匹配 "一体化运维"、"运维自动化"

* 监控 自动匹配 "监控告警"、"告警"

* CMDB 自动匹配 "配置管理"、"资产管理"

### 4. 案例价值分析

**自动分析内容**:

* 价格范围统计（最小值、最大值、中位数）

* 成功模式总结（共性特征）

* 差异化要点识别（独特优势）

* 时间分布分析

## Instructions

### 执行角色与核心原则

**你的角色定位**：
你是一位专业的案例分析师，擅长从大量案例中快速找到最相关的成功案例。

**你的任务目标**：
基于客户的行业、关注模块、项目特点，从案例库中智能筛选和推荐最匹配的案例。

**专业能力**：

* 深刻理解不同行业的业务特点和痛点

* 精通IT解决方案的各类模块和技术栈

* 善于识别案例之间的相似性和差异性

* 理解销售场景中案例的价值和使用方式

**案例匹配原则**：

1. **真实性第一**：严格基于案例库数据，严禁虚构案例
2. **相关性优先**：优先匹配行业和模块都相符的案例
3. **时效性考虑**：优先推荐近期案例，增强说服力
4. **灵活匹配**：支持行业回退和模块同义词匹配，提高匹配率
5. **完整返回**：返回所有匹配案例（不进行精选或截取），避免遗漏有价值的案例

### 步骤 1: 读取案例库

**读取案例库文件**:

```
使用 Read 工具读取 case_library_path 指定的案例库文件
默认路径: sale-kb/行业案例/公司案例库（持续更新）.md
```

**⚠️ 重要：路径处理**:

* **默认路径**：`sale-kb/行业案例/公司案例库（持续更新）.md`（相对于工作空间根目录）

* **知识库维护**：此文件由知识库统一维护，持续更新，无需手动指定路径

* **如果读取失败**：

  * 方法 1：尝试使用 Bash 工具动态查找：`find . -name "公司案例库*" -path "*/行业案例/*" -print -quit`

  * 方法 2：提示调用方检查知识库目录是否存在

  * 方法 3：返回明确错误信息，说明文件路径不正确

**解析案例库格式**:
案例库为 Markdown 表格格式：

```markdown
| 签约日期 | 客户名称 | 销售机会 | 合同名称 | 行业 | 合同金额 |
|---------|---------|---------|---------|------|---------|
| 2021/01/04 | 延锋汽车饰件系统有限公司 | 延锋汽车CMDB | IT产品采购合同 | 制造业 | 1,297,500 |
```

**⚠️ 关键：列映射关系（新格式 vs 旧格式）**:

| 新知识库列名 | 对应旧字段    | 说明                                |
| ------ | -------- | --------------------------------- |
| 签约日期   | 合同时间     | 格式从"2018年10月"变为"2021/01/04"       |
| 客户名称   | 客户名称     | 不变                                |
| 销售机会   | 模块（替代）   | ⚠️ 无独立模块列，需从此字段+合同名称提取模块信息        |
| 合同名称   | 合同名称     | 不变                                |
| 行业     | 行业细分（替代） | ⚠️ 单级行业分类（如"城市商业银行"），无"一级-二级"格式   |
| 合同金额   | 合同金额(万)  | ⚠️ 单位为元（如1,297,500），需除以10000转换为万元 |

**解析要点**:

* 识别表格列名（6列：签约日期、客户名称、销售机会、合同名称、行业、合同金额）

* **模块提取**：从"销售机会"和"合同名称"两个字段中提取模块关键词（CMDB、ITSM/ITSC、DevOps、自动化、监控、人行上报等）

* **金额转换**：去除千位逗号（1,297,500 → 1297500），再除以10000转换为万元（129.75万）

* **日期解析**：将"2021/01/04"解析为标准日期格式

**模块关键词提取规则**（从"销售机会"和"合同名称"中提取）:

```
优先从"销售机会"字段提取，补充从"合同名称"字段提取：

关键词匹配规则（按优先级）：
1. "CMDB" → 模块: CMDB
2. "ITSM" 或 "ITSC" 或 "IT服务管理" → 模块: ITSM
3. "DevOps" 或 "devops" 或 "持续交付" 或 "持续集成" 或 "持续部署" → 模块: DevOps
4. "自动化" → 模块: 自动化
5. "监控" → 模块: 监控
6. "人行" 或 "金融业科技信息" 或 "元数据上报" → 模块: 人行上报
7. "低代码" → 模块: 低代码
8. "容器" 或 "K8s" 或 "Kubernetes" → 模块: 容器化
9. "安全" → 模块: 安全
10. "全平台" 或 "平台租用" → 模块: 全平台
11. "Agent" 或 "agent" → 模块: Agent

15. "扩容" → 模块: 扩容

一条案例可以匹配多个模块关键词，记录所有匹配的模块（如"CMDB+ITSM"）
如果无法识别任何模块关键词，标记为"其他"
```

**错误处理**:

* 如果文件不存在，返回错误信息

* 如果文件格式不正确，尝试解析或返回错误

* 如果案例库为空，返回空结果

### 步骤 2: 多维度筛选

#### 2.1 行业筛选

**⚠️ 重要：行业格式变化**

* **旧格式**："行业细分"为二级格式（如"金融-银行"、"制造-汽车"）

* **新格式**："行业"为单级格式（如"城市商业银行"、"券商"、"制造业"），行业分类更细

**行业分组映射**（用于支持上级行业匹配和模糊匹配）:

```
金融 → [城市商业银行, 农村商业银行, 股份制银行, 国有银行, 外资银行, 券商, 基金, 保险, 期货, 消费金融, 其他金融]
银行 → [城市商业银行, 农村商业银行, 股份制银行, 国有银行, 外资银行]
金融-银行 → [城市商业银行, 农村商业银行, 股份制银行, 国有银行, 外资银行]  // 兼容旧参数格式
金融-券商 → [券商]  // 兼容旧参数格式
金融-保险 → [保险]  // 兼容旧参数格式
金融-基金 → [基金]  // 兼容旧参数格式
制造业 → [制造业]
制造-汽车 → [制造业]  // 兼容旧参数格式
制造-化工 → [制造业]  // 兼容旧参数格式
互联网 → [互联网]
教育 → [教育]
运营商 → [运营商]
政府 → [政府]
能源电力 → [能源电力]
```

**筛选逻辑**:

```
如果 target_industry 参数提供：
  1. 优先精确匹配"行业"字段等于 target_industry 的案例
  2. 如果精确匹配无结果：
     a. 查找行业分组映射表，看 target_industry 是否为分组名称
        例如: target_industry="银行" → 匹配 [城市商业银行, 农村商业银行, 股份制银行, 国有银行, 外资银行]
        例如: target_industry="金融" → 匹配所有金融子行业
     b. 如果 target_industry 包含"-"（旧格式如"金融-银行"），提取"-"后的子分类进行映射
     c. 如果仍无结果，进行模糊匹配（目标行业是案例行业的子串，或案例行业包含目标行业关键词）
  3. 如果精确匹配有结果，仍建议返回分组映射后的扩展结果（提示用户可能有更多相关案例）

如果 target_industry 未提供：
  - 跳过行业筛选，保留所有案例
```

**行业匹配示例**（基于新知识库格式）:

* 目标行业: "城市商业银行" → 精确匹配所有"城市商业银行"案例

* 目标行业: "银行" → 通过分组映射匹配 \[城市商业银行, 农村商业银行, 股份制银行, 国有银行, 外资银行]

* 目标行业: "金融" → 通过分组映射匹配所有金融子行业（银行类+券商+基金+保险+期货+消费金融+其他金融）

* 目标行业: "金融-银行"（旧格式兼容）→ 通过"-"分割提取"银行"，再分组映射

* 目标行业: "制造业" → 精确匹配"制造业"

* 目标行业: "制造-汽车"（旧格式兼容）→ 提取"制造"前缀无映射，尝试提取"汽车"子串模糊匹配"制造业"

#### 2.2 模块筛选

**⚠️ 重要：模块来源变化**

* **旧格式**：模块信息存储在独立的"模块"列

* **新格式**：模块信息从"销售机会"和"合同名称"两个字段中提取（见步骤1的模块关键词提取规则）

**筛选逻辑**:

```
如果 target_modules 参数提供：
  - 在步骤1中已为每条案例提取了模块列表（从"销售机会"+"合同名称"字段）
  - 筛选案例的模块列表中包含任一 target_modules 的案例
  - 支持模糊匹配、多关键词匹配和同义词匹配

如果 target_modules 未提供：
  - 跳过模块筛选，保留所有案例
```

**模块同义词映射**:

```
ITSM → ["ITSM", "ITSC", "IT服务管理", "流程平台", "流程管理", "服务管理"]
自动化 → ["自动化", "一体化运维", "运维自动化", "自动化运维"]
监控 → ["监控", "监控告警", "告警", "智能监控"]
CMDB → ["CMDB", "配置管理", "资产管理", "CDMB"]  // CDMB 为常见拼写错误
DevOps → ["DevOps", "devops", "持续交付", "持续集成", "持续部署", "CI/CD"]
人行上报 → ["人行", "金融业科技信息", "元数据上报", "科技信息综合管理", "上报"]
低代码 → ["低代码", "低代码开发"]
```

**模块匹配规则**:

* 单模块匹配: 案例模块列表包含目标模块关键词或其同义词

* 多模块匹配: 案例模块列表包含任一目标模块或其同义词即可

* 模糊匹配: 关键词子串匹配（如"CMDB"可匹配提取结果中的"CMDB"标签）

* 同义词匹配: "ITSM"可匹配提取结果中的"ITSC"标签

**示例**（基于新知识库格式）:

```
目标模块: ["CMDB", "监控"]

案例1: 销售机会="深圳证券交易所CMDB3期项目", 合同名称="XX采购合同"
  → 提取模块: ["CMDB"] → 匹配

案例2: 销售机会="华泰保险CMDB配置管理系统", 合同名称="XX合同"
  → 提取模块: ["CMDB"] → 匹配

案例3: 销售机会="2025LVMH-CMDB&ITSM五期项目", 合同名称="XX合同"
  → 提取模块: ["CMDB", "ITSM"] → 同时匹配两个目标

案例4: 销售机会="联通自动化运维", 合同名称="XX合同"
  → 提取模块: ["自动化"] → 不匹配（目标不含自动化）

目标模块: ["ITSM"]
案例5: 销售机会="微创ITSM", 合同名称="EasyOps全栈运维CMDB+ITSC平台"
  → 提取模块: ["ITSM", "ITSC"] → 匹配（合同名称中也有"ITSC"=ITSM同义词）

目标模块: ["自动化"]
案例6: 销售机会="名创优品持续部署平台项目", 合同名称="XX合同"
  → 提取模块: ["DevOps"] → 不匹配（持续部署=DevOps，非自动化）
```

#### 2.3 时间筛选

**时间范围策略**:

**auto 策略（默认）**:

```
1. 优先筛选一年内案例
2. 如果一年内案例少于3个，扩展到五年内
3. 如果五年内案例仍少于3个，使用全部案例
```

**指定年限策略**:

```
1year: 只返回一年内案例
3years: 只返回三年内案例
5years: 只返回五年内案例
all: 返回所有案例，不限时间
```

**时间计算**:

* 当前时间: 使用执行时的实际日期

* 一年内: 当前时间 - 365天

* 三年内: 当前时间 - 1095天

* 五年内: 当前时间 - 1825天

#### 2.4 金额范围筛选

**⚠️ 金额单位转换**:

* **新知识库格式**：合同金额单位为**元**（如 1,297,500），需除以10000转换为**万元**（129.75万）

* **转换规则**：去除千位逗号 → 数值÷10000 → 保留两位小数

**筛选逻辑**:

```
如果 min_amount 参数提供：
  - 筛选转换后的合同金额（万元）>= min_amount 的案例
  - 金额单位统一为万元

如果 min_amount 未提供或为 0：
  - 跳过金额筛选
```

### 步骤 3: 相似度计算与排序

#### 3.1 相似度计算

**计算公式**:

```
相似度分数 = 行业匹配分 + 模块匹配分 + 时间分 + 金额分

行业匹配分:
  - 行业完全匹配（如目标"城市商业银行"，案例"城市商业银行"）: +40分
  - 行业分组映射匹配（如目标"银行"，案例"城市商业银行"）: +35分
  - 行业大类匹配（如目标"金融"，案例"券商"）: +20分
  - 旧格式兼容匹配（如目标"金融-银行"，案例"城市商业银行"）: +35分
  - 行业模糊匹配（子串匹配）: +10分
  - 行业不匹配: 0分

模块匹配分:
  - 模块完全匹配（从销售机会/合同名称中提取到的模块与目标一致）: +30分
  - 模块同义词匹配（如目标"ITSM"，提取结果含"ITSC"）: +25分
  - 模块部分匹配（目标模块是提取结果的一部分）: +15分
  - 模块不匹配: 0分

时间分:
  - 一年内: +20分
  - 一到三年: +10分
```

````json
### 示例 1: 基础案例匹配（指定项目名称）

```json
````

### 示例 3: 行业分组映射匹配

**输入**:

```json
{
  "target_industry": "银行",
  "target_modules": ["ITSM"],
  "time_range": "auto"
}
```

**处理流程**:

1. 目标"银行"通过分组映射匹配 \[城市商业银行, 农村商业银行, 股份制银行, 国有银行, 外资银行] → 找到所有银行类案例
2. 从"销售机会"和"合同名称"中筛选包含"ITSM"或其同义词"ITSC"、"IT服务管理"的案例
3. 按时间筛选（auto策略）
4. 计算相似度（分组映射匹配得35分，同义词匹配得25分）
5. 返回所有匹配的案例
6. 保存案例匹配报告（或自动检测的项目）

**输出价值**：

* 通过行业分组映射，"银行"可匹配所有银行子类型（城商行、农商行、股份制银行等），提高匹配覆盖率

* 同义词匹配确保"ITSC"案例也被识别为ITSM相关案例

* 提高匹配成功率，避免因行业分类粒度不同或模块名称不同导致无匹配结果

* 返回所有匹配案例，不遗漏任何有价值的参考

* 自动生成报告文件供后续查阅

```markdown

| 2021/08/13 | 贵阳银行股份有限公司 | 贵阳银行统一运维平台 | 统一运维平台产品及服务采购合同 | 城市商业银行 | 3,840,000 |
```

**文件路径**: `sale-kb/行业案例/公司案例库（持续更新）.md`

**字段说明**:

| 字段   | 说明                   | 示例            |
| ---- | -------------------- | ------------- |
| 签约日期 | 合同签约日期，格式 YYYY/MM/DD | 2021/01/04    |
| 客户名称 | 客户全称                 | 延锋汽车饰件系统有限公司  |
| 销售机会 | 项目简要描述，**包含模块关键词**   | 延锋汽车CMDB      |
| 合同名称 | 合同正式名称               | IT产品采购合同      |
| 行业   | **单级行业分类**（无二级格式）    | 制造业、城市商业银行、券商 |
| 合同金额 | **单位为元**（需÷10000转万元） | 1,297,500     |

1. **无独立"模块"列** → 模块信息需从"销售机会"和"合同名称"字段中提取
2. **行业为单级分类** → 通过行业分组映射表支持上级行业匹配
3. **合同金额单位为元** → 需除以10000转换为万元
4. **签约日期格式不同** → "2021/01/04" 而非 "2024年3月"

# Case Matching (案例智能匹配技能)

## Purpose

为销售人员智能匹配最相关的成功案例，基于行业、模块、时间等多维度进行筛选，提供量化的匹配度分析和案例推荐。

**核心功能**:
- 多维度案例筛选（行业+模块+时间）
- 行业回退匹配（细分行业无结果时回退到一级行业）
- 模块同义词智能匹配（ITSM↔流程平台、自动化↔一体化运维）
- 智能相似度计算和排序
- 案例价值分析（价格范围、成功模式）
- 返回所有匹配案例（不进行精选或截取）

**核心价值**:
- ✅ **精准匹配**：多维度筛选找到最相关案例
- ✅ **智能排序**：按相似度和时间自动排序
- ✅ **灵活匹配**：支持行业回退和模块同义词匹配
- ✅ **数据驱动**：基于真实案例库，严禁虚构
- ✅ **高效复用**：统一算法，多个Agent可调用
- ✅ **完整性**：返回所有匹配案例，不进行精选或截取，避免遗漏

## When to Use

在以下情况下使用此技能：
- 销售准备阶段需要找同行业成功案例
- 需求分析时需要参考类似项目
- 会议分析后需要匹配相关案例
- 方案呈现时需要案例背书
- 招投标时需要业绩证明

## Capabilities

### 1. 多维度案例筛选

**支持的筛选维度**:
- **行业筛选**: 按行业细分精确匹配（如"金融-银行"、"金融-券商"、"制造-化工"），支持行业回退（如"制造业-化工"无匹配时回退到"制造业"）
- **模块筛选**: 支持模糊匹配和同义词匹配（CMDB、ITSM/流程平台、一体化运维/自动化、监控、DevOps等）
- **时间筛选**: 支持一年内、三年内、五年内、全部等策略
- **金额范围**: 支持按合同金额范围筛选

### 2. 智能相似度计算

**相似度计算因素**:
- 行业完全匹配 +40分
- 模块完全匹配 +30分
- 时间越近分数越高（最近一年 +20分，一到三年 +10分）
- 合同金额相近 +10分

### 3. 智能匹配增强

**行业回退匹配**:
- 优先精确匹配细分行业（如"制造业-化工"）
- 无结果时自动回退到一级行业（如"制造业"）
- 提高匹配成功率，避免因行业细分过细导致无匹配结果

**模块同义词匹配**:
- ITSM 自动匹配 "流程平台"、"流程管理"
- 自动化 自动匹配 "一体化运维"、"运维自动化"
- 监控 自动匹配 "监控告警"、"告警"
- CMDB 自动匹配 "配置管理"、"资产管理"

### 4. 案例价值分析

**自动分析内容**:
- 价格范围统计（最小值、最大值、中位数）
- 成功模式总结（共性特征）
- 差异化要点识别（独特优势）
- 时间分布分析

## Parameters

| 参数 | 类型 | 必须 | 默认值 | 描述 |
|------|------|------|--------|------|
| case_library_path | string | ✅ | - | 案例库文件路径（推荐使用：`/plugins/*sales*/skills/company-research/resource/data/caseLibrary.md` 或使用 Bash 动态查找：`find /plugins -name "caseLibrary.md" -path "*/*sales*/*" -print -quit`） |
| project_name | string | ❌ | auto | 项目名称，用于确定案例匹配报告的保存路径（如"XX银行CMDB项目"）。如不指定，系统将自动检测：若只有一个项目则自动使用，若有多个项目则询问用户选择 |
| target_industry | string | ❌ | - | 目标行业（如"金融-银行"），如不指定则不按行业筛选 |
| target_modules | array | ❌ | [] | 目标模块列表（如["CMDB", "监控"]） |
| time_range | string | ❌ | auto | 时间范围策略（1year/3years/5years/all/auto） |
| max_results | number | ❌ | 999 | 最多返回案例数量（999表示不限制，返回所有匹配案例。不推荐修改此参数，以免遗漏有价值的案例） |
| min_amount | number | ❌ | 0 | 最小合同金额（万元） |
| sort_by | string | ❌ | similarity | 排序方式（similarity/time/amount） |

## Instructions

### 执行角色与核心原则

**你的角色定位**：
你是一位专业的案例分析师，擅长从大量案例中快速找到最相关的成功案例。

**你的任务目标**：
基于客户的行业、关注模块、项目特点，从案例库中智能筛选和推荐最匹配的案例。

**专业能力**：
- 深刻理解不同行业的业务特点和痛点
- 精通IT解决方案的各类模块和技术栈
- 善于识别案例之间的相似性和差异性
- 理解销售场景中案例的价值和使用方式

**案例匹配原则**：
1. **真实性第一**：严格基于案例库数据，严禁虚构案例
2. **相关性优先**：优先匹配行业和模块都相符的案例
3. **时效性考虑**：优先推荐近期案例，增强说服力
4. **灵活匹配**：支持行业回退和模块同义词匹配，提高匹配率
5. **完整返回**：返回所有匹配案例（不进行精选或截取），避免遗漏有价值的案例

### 步骤 1: 读取案例库

**读取案例库文件**:
```
使用 Read 工具读取 case_library_path 指定的案例库文件
```

**⚠️ 重要：路径处理**:
- **推荐方式 1**：使用 Read 工具读取：`/plugins/*sales*/skills/company-research/resource/data/caseLibrary.md`
- **推荐方式 2**：使用 Bash 动态查找：
  ```bash
  find /plugins -name "caseLibrary.md" -path "*/*sales*/*" -print -quit
  ```
- **如果相对路径读取失败**：
  - 方法 1：提示调用方使用 Bash 工具动态查找文件位置
  - 方法 2：返回明确错误信息，说明文件路径不正确

**解析案例库格式**:
案例库通常为 Markdown 表格格式：
```markdown
| 客户名称 | 行业细分 | 模块 | 合同名称 | 合同金额(万) | 合同时间 |
|---------|---------|------|---------|-------------|---------|
| 中国银行 | 金融-银行 | CMDB | XX项目 | 50 | 2024-03 |
```

**解析要点**:
- 识别表格列名（支持多种变体）
- 提取每行案例数据
- 处理金额格式（支持"50万"、"50"、"50.5"等）
- 处理日期格式（支持"2024-03"、"2024年3月"等）

**错误处理**:
- 如果文件不存在，返回错误信息
- 如果文件格式不正确，尝试解析或返回错误
- 如果案例库为空，返回空结果

### 步骤 2: 多维度筛选

#### 2.1 行业筛选

**筛选逻辑**:
```
如果 target_industry 参数提供：
  1. 优先筛选"行业细分"字段完全匹配 target_industry 的案例
  2. 如果完全匹配无结果且 target_industry 包含"-"（如"制造业-化工"）：
     - 提取一级行业（如"制造业"）
     - 回退筛选：匹配所有以该一级行业开头的案例
  3. 支持精确匹配（如"金融-银行"）

如果 target_industry 未提供：
  - 跳过行业筛选，保留所有案例
```

**行业匹配示例**:
- 目标行业: "金融-银行" → 匹配"金融-银行"，不匹配"金融-保险"
- 目标行业: "制造业-化工" → 优先匹配"制造业-化工"，若无结果则回退匹配"制造业-*"（制造业-化工、制造业-汽车等）
- 目标行业: "制造业" → 匹配"制造业-化工"、"制造业-汽车"等所有制造业案例

#### 2.2 模块筛选

**筛选逻辑**:
```
如果 target_modules 参数提供：
  - 筛选"模块"字段包含任一 target_modules 的案例
  - 支持模糊匹配、多关键词匹配和同义词匹配

如果 target_modules 未提供：
  - 跳过模块筛选，保留所有案例
```

**模块同义词映射**:
```
ITSM → ["ITSM", "流程平台", "流程管理"]
自动化 → ["自动化", "一体化运维", "运维自动化", "自动化运维"]
监控 → ["监控", "监控告警", "告警"]
CMDB → ["CMDB", "配置管理", "资产管理"]
```

**模块匹配规则**:
- 单模块匹配: 案例模块包含目标模块关键词或其同义词
- 多模块匹配: 案例模块包含任一目标模块或其同义词即可
- 模糊匹配: "CMDB" 可匹配 "CMDB配置管理"、"CMDB+监控"等
- 同义词匹配: "ITSM" 可匹配 "流程平台"、"流程管理"等

**示例**:
```
目标模块: ["CMDB", "监控"]
案例1: 模块="CMDB" → 匹配
案例2: 模块="监控告警" → 匹配（监控的同义词）
案例3: 模块="CMDB+监控" → 匹配（同时匹配两个关键词）
案例4: 模块="自动化" → 不匹配

目标模块: ["ITSM"]
案例1: 模块="ITSM" → 匹配
案例2: 模块="流程平台" → 匹配（ITSM的同义词）
案例3: 模块="流程管理平台" → 匹配（包含同义词"流程平台"）

目标模块: ["自动化"]
案例1: 模块="自动化" → 匹配
案例2: 模块="一体化运维" → 匹配（自动化的同义词）
案例3: 模块="运维自动化平台" → 匹配（包含同义词"运维自动化"）
```

#### 2.3 时间筛选

**时间范围策略**:

**auto 策略（默认）**:
```
1. 优先筛选一年内案例
2. 如果一年内案例少于3个，扩展到五年内
3. 如果五年内案例仍少于3个，使用全部案例
```

**指定年限策略**:
```
1year: 只返回一年内案例
3years: 只返回三年内案例
5years: 只返回五年内案例
all: 返回所有案例，不限时间
```

**时间计算**:
- 当前时间: 使用执行时的实际日期
- 一年内: 当前时间 - 365天
- 三年内: 当前时间 - 1095天
- 五年内: 当前时间 - 1825天

#### 2.4 金额范围筛选

**筛选逻辑**:
```
如果 min_amount 参数提供：
  - 筛选合同金额 >= min_amount 的案例
  - 金额单位统一为万元

如果 min_amount 未提供或为 0：
  - 跳过金额筛选
```

### 步骤 3: 相似度计算与排序

#### 3.1 相似度计算

**计算公式**:
```
相似度分数 = 行业匹配分 + 模块匹配分 + 时间分 + 金额分

行业匹配分:
  - 行业完全匹配（如目标"金融-银行"，案例"金融-银行"）: +40分
  - 行业回退匹配（如目标"制造业-化工"，案例"制造业-汽车"）: +20分
  - 行业不匹配: 0分

模块匹配分:
  - 模块完全匹配: +30分
  - 模块同义词匹配（如目标"ITSM"，案例"流程平台"）: +25分
  - 模块部分匹配: +15分
  - 模块不匹配: 0分

时间分:
  - 一年内: +20分
  - 一到三年: +10分
  - 三到五年: +5分
  - 五年以上: 0分

金额分:
  - 金额相近（差距小于20%）: +10分
  - 金额差距较大: 0分
```

**示例计算**:
```
目标: 金融-银行, CMDB, 2024年
案例A: 金融-银行, CMDB, 2023-08, 50万
  - 行业匹配: 40分（完全匹配）
  - 模块匹配: 30分（完全匹配）
  - 时间: 20分（一年内）
  - 金额: 10分
  - 总分: 100分

案例B: 金融-保险, CMDB, 2021-03, 45万
  - 行业匹配: 0分（不匹配）
  - 模块匹配: 30分（完全匹配）
  - 时间: 5分（三到五年）
  - 金额: 10分
  - 总分: 45分

目标: 制造业-化工, ITSM, 2024年（假设"制造业-化工"无匹配，回退到"制造业"）
案例C: 制造业-汽车, 流程平台, 2023-05, 80万
  - 行业匹配: 20分（回退匹配）
  - 模块匹配: 25分（同义词匹配，ITSM↔流程平台）
  - 时间: 20分（一年内）
  - 金额: 0分
  - 总分: 65分

案例D: 制造业-电子, 一体化运维, 2022-11, 60万
  - 行业匹配: 20分（回退匹配）
  - 模块匹配: 0分（不匹配，目标是ITSM）
  - 时间: 10分（一到三年）
  - 金额: 0分
  - 总分: 30分
```

#### 3.2 排序

**排序方式**:

**similarity (默认)**:
- 按相似度分数降序排列
- 相似度相同时，按时间降序（最新优先）

**time**:
- 按合同时间降序排列（最新优先）

**amount**:
- 按合同金额降序排列（金额最大优先）

#### 3.3 返回所有匹配案例

**返回逻辑**:
```
默认返回所有匹配的案例（max_results = 999，不限制数量）
不进行精选或截取，确保所有匹配案例都被包含在结果中
如果调用方明确指定了 max_results 参数，则只返回前 N 个案例（不推荐）
```

### 步骤 4: 案例价值分析

**统计分析内容**:

#### 4.1 价格范围统计

**计算指标**:
- 最小金额: min(所有匹配案例的金额)
- 最大金额: max(所有匹配案例的金额)
- 平均金额: avg(所有匹配案例的金额)
- 中位数金额: median(所有匹配案例的金额)

**示例**:
```
匹配到5个案例: 50万, 80万, 60万, 120万, 45万
- 最小金额: 45万
- 最大金额: 120万
- 平均金额: 71万
- 中位数: 60万
```

#### 4.2 时间分布分析

**统计维度**:
- 一年内案例数量和占比
- 一到五年案例数量和占比
- 五年以上案例数量和占比

**示例**:
```
共匹配15个案例:
- 一年内: 8个 (53%)
- 一到五年: 5个 (33%)
- 五年以上: 2个 (14%)
```

#### 4.3 成功模式总结

**分析方法**:
- 识别高频行业（出现最多的行业）
- 识别高频模块（出现最多的模块组合）
- 识别典型客户类型（国企、民企、外企）

**示例**:
```
成功模式:
- 主要集中在金融-银行行业（60%）
- 以CMDB+监控组合为主（70%）
- 典型合同金额范围: 50-120万
```

#### 4.4 差异化要点

**识别方法**:
- 找出独特的模块组合
- 找出高价值大单
- 找出特殊行业案例

**示例**:
```
差异化要点:
- XX银行120万大单（CMDB+自动化+监控全栈方案）
- 首个保险行业CMDB案例（XX保险）
- 最快交付案例（XX证券，2个月完成）
```

### 步骤 7: 更新项目状态（记录会话轨迹）

**在案例匹配完成后，记录到项目状态文件的会话轨迹**:

```javascript
// 构建会话轨迹摘要

    "operation": "append",
    "agent_name": "case-matching"
  }
)
```

**会话轨迹摘要示例**:
- **有行业有模块**: "案例匹配：为金融-银行的CMDB+监控项目匹配到8个案例，价格范围45-120万"
- **无行业有模块**: "案例匹配：为跨行业的DevOps项目匹配到15个案例，价格范围50-200万"

- **无匹配结果**: "案例匹配：未找到符合条件的案例（教育-高校，区块链）"

**更新逻辑**:
1. 只有在 status = "success" 且有匹配结果时才记录会话轨迹
2. 如果 status = "no_match"，可选择性记录（记录为"未找到符合条件的案例"）
3. 如果 status = "error"，不记录会话轨迹
4. 摘要应简洁明了，突出关键信息（目标条件+匹配数量+价格范围）

**注意事项**:
- project_name 必须已确定（通过用户提供、自动检测或用户选择）
- 会话轨迹记录应在文件保存后进行（确保报告已生成）
- 如果项目状态更新失败，不影响案例匹配的主要功能（可记录日志但不中断流程）

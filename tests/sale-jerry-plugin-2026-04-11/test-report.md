#### 问题 #3：project-init 步骤2.5 中 migrate_dir() 函数不够健壮
**文件**：`project-init/SKILL.md` 第339-358行

**问题描述**：
```bash
rmdir "${OLD_PATH}" 2>/dev/null || echo "⚠️ 旧目录非空..."
```

当旧目录非空且 rmdir 失败时，脚本仅打印警告，继续执行。这可能导致：
1. 旧目录未被完全删除
2. 后续步骤混淆新旧目录

**影响**：
- 迁移后项目目录结构可能混乱
- 下游 Skill 访问文件时可能找到重复目录

**修复建议**：
```bash
migrate_dir() {
  local OLD_NAME="$1"
  local NEW_NAME="$2"
  local OLD_PATH="${PROJECT_ROOT}/${OLD_NAME}"
  local NEW_PATH="${PHASE_DIR}/${NEW_NAME}"

  if test -d "${OLD_PATH}"; then
    mkdir -p "${NEW_PATH}"
    if [ "$(ls -A "${OLD_PATH}" 2>/dev/null)" ]; then
      # 非空目录，迁移文件
      mv "${OLD_PATH}"/* "${NEW_PATH}/" 2>/dev/null || true
      echo "✅ 迁移: ${OLD_NAME}/ → 售前阶段/${NEW_NAME}/ (${count}个文件)"
    fi
    # 删除旧目录前检查是否为空
    if rmdir "${OLD_PATH}" 2>/dev/null; then
      echo "✅ 删除旧目录: ${OLD_NAME}/"
    else
      echo "❌ 无法删除旧目录 ${OLD_NAME}/（目录非空或权限不足）"
      return 1  # 返回错误，中止迁移
    fi
  fi
}
```

**严重级别**：高（影响迁移数据完整性）

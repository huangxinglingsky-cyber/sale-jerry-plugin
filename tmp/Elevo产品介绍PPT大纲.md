## 第二部分：Elevo 是什么（3-4 页）

### Slide 4：Elevo 产品定位

> **一句话定位**：Elevo（Knodo）是一个 AI 驱动的先进生产力平台，让企业通过 AI Agent 团队自动化处理复杂的知识工作。

#### 官方定义（来源：knodo.vip/docs）

Knodo 是一个 AI 驱动的先进生产力平台，旨在帮助团队更高效地完成知识工作。通过整合 **AI 助手、知识库管理和团队协作** 功能，Knodo 让团队能够：
- 与 AI 助手进行智能对话，获取专业建议和帮助
- 集中管理团队知识，支持文档版本控制
- 组建 AI Agent 团队，自动化处理复杂任务
- 任务管理，任务可委派给 Agent 执行

#### 产品全景图

```
Elevo（Knodo）平台
├── 组织管理 — 团队、成员、权限
├── 工作空间 — 项目协作单元
│   ├── 知识库 — 文档管理、版本控制、共享
│   ├── AI 助手 — 对话、Agent 团队、模型配置
│   ├── 任务管理 — 任务创建、定时执行、事件回调
│   └── 扩展能力 — 技能市场、插件管理
├── 记忆系统 — AI 记忆管理、自动记忆提取
└── 多端接入 — Web、钉钉、飞书、企业微信
```

### Slide 5：Elevo 的核心概念层级

> **通俗理解**：Elevo 是一栋大楼，组织是公司，工作空间是部门，知识库是资料室，Agent 是员工，Skill 是技能，MCP 是外部工具箱。

#### 五层架构关系

```
第 1 层：组织（Organization）
  └── 第 2 层：工作空间（Workspace）
       ├── 第 3 层：知识库（Knowledge Base）— 文档、案例、模板
       ├── 第 4 层：AI 配置
       │    ├── Agent 团队（智能体）— 具有特定角色和能力的 AI 员工
       │    ├── Skill（技能）— Agent 可以执行的具体操作
       │    ├── MCP（模型上下文协议）— 连接外部系统的通道
       │    └── 模型设置（LLM）— 大脑引擎
       └── 第 5 层：成员与权限 — 人 + AI 协同工作
```

### Slide 6：Elevo vs 传统工具 — 为什么不一样？

| 对比维度 | 传统协作工具（钉钉/飞书/企微） | 传统 AI（ChatGPT/Claude 直接用） | **Elevo（Knodo）** |
|---------|--------------------------|----------------------------|------------------|
| 知识管理 | 文件散落在各处 | 无持久知识 | 工作空间内置知识库 + Git 版本控制 |
| AI 能力 | 通用对话，不懂业务 | 通用 AI，无专业知识 | **Agent 团队 + Skill + 知识库**，懂行业、懂产品、懂流程 |
| 团队协作 | 人找人 | 一人一号，知识不共享 | 人 + AI 协同，知识沉淀在工作空间 |
| 业务流程 | 靠人记忆和习惯 | 无流程概念 | **Agent 自动编排流程**（项目初始化→销售准备→需求分析→投标） |
| 数据安全 | 公有云，数据不可控 | 对话记录在第三方 | 私有化部署、组织级隔离、权限管控 |
| 可扩展性 | 固定功能 | 固定能力 | **插件 + 技能市场**，持续扩展 |

### Slide 8：技术视角 — 架构全景

#### 大模型（LLM）— 大脑引擎

> **技术定义**：大模型是 AI 的基础推理引擎，负责理解自然语言、生成文本、做出决策。
>
> **Elevo 支持**：Claude（Anthropic）、通义千问（阿里云）、DeepSeek、豆包（字节跳动）、Gemini（Google）、OpenAI GPT 等主流模型。
>
> **角色比喻**：如果 Elevo 是一台电脑，大模型就是 CPU — 它处理所有"思考"工作。

- Elevo 不绑定单一模型，可按场景灵活切换（日常用 Claude Sonnet，复杂任务用 Claude Opus，简单任务用 Haiku 省成本）
- 模型选择在工作空间级别配置，不同工作空间可以用不同模型

#### MCP（Model Context Protocol）— 外部连接协议

> **技术定义**：MCP 是一个标准化协议，让 AI 能够安全地连接和调用外部系统（数据库、API、文件系统等），为 AI 提供上下文数据。
>
> **角色比喻**：MCP 是电脑的 USB 接口 — 标准化地连接各种外部设备。

- 在 Elevo 中，MCP 通过 `.mcp.json` 配置，每个插件可以定义自己的 MCP 服务器
- 举例：`claude-mem` MCP 让 AI 能搜索跨会话记忆数据库；`web-reader` MCP 让 AI 能读取网页内容

#### Agent（智能体）— 业务角色

> **技术定义**：Agent 是经过特定配置的 AI 助手，通过 Prompt 定义行为，专注于特定领域或任务，可与其他 Agent 协同工作。
>
> **官方定义（来源：knodo.vip/docs/ai-assistant/ai-agents）**：
> - **专业化**：专注于特定领域或任务
> - **可配置**：通过 Prompt 定义行为
> - **可协作**：多个 Agent 可以协同工作
>
> **角色比喻**：Agent 是公司里不同岗位的专业员工 — 有销售经理、有售前工程师、有文档专员。

- **系统 Agent**：平台预置，开箱即用（通用助手、代码专家、文档写手、数据分析师）
- **插件 Agent**：通过插件安装（如我们的 `sales-master` 销售主控代理）
- **每个 Agent 都有**：角色定义（Prompt）、能力描述、可调用的 Skill 列表、独立的对话上下文

#### Skill（技能）— 具体操作能力

> **技术定义**：Skill 是 Agent 可以执行的具体操作单元，以 SKILL.md 文件定义，包含触发条件、执行步骤、参数说明和示例。
>
> **角色比喻**：Skill 是员工的专业技能证书 — 员工（Agent）根据需要调用不同的技能（Skill）来完成工作。

- 每个 Skill 是一个独立目录：`SKILL.md`（核心指令） + `templates/`（模板） + `examples/`（示例） + `references/`（参考资料） + `scripts/`（脚本）
- Skill 通过参数触发，支持文件操作、API 调用、数据处理等
- 举例：`sales-prep`（销售准备）、`bid-analysis`（招投标分析）、`requirement-matching`（需求匹配）

#### Plugin（插件）— 能力扩展包

> **技术定义**：插件是包含 Agent 定义和工具函数的扩展包，是将多个 Agent、Skill、MCP、Hook 打包分发的载体。
>
> **角色比喻**：插件就是一个"部门"或"事业部" — 包含一组有协作关系的员工（Agent）、工具（MCP）和工作流程（Skill）。

```
sale-jerry-plugin/
├── agents/          → Agent 定义（销售主控、售前专家等）
├── skills/          → 32 个 Skill（企业调研、案例匹配、话术生成等）
├── .mcp.json        → MCP 配置（记忆搜索、网页读取等）
├── hooks/           → 事件钩子（自动通知等）
└── .claude-plugin/  → 插件清单
```

### Slide 9：拟人场景视角 — 把 Elevo 想象成一家公司

> **核心比喻**：Elevo 就是一家"AI 公司"，你是老板，工作空间是项目组。

#### 大模型 = 公司的大脑/学历

- 大模型决定了 AI 员工的"智商"
- 用 Claude Opus = 招了一个博士；用 Haiku = 招了一个实习生
- Elevo 的好处：**你可以随时换大脑**，不同项目用不同级别的 AI
- 销售理解：就像你招人，简单的事让新人做，复杂的事让老手做

#### MCP = 公司的外部合作渠道

- MCP 是公司和外部世界沟通的标准化接口
- 比如：连接企业数据库查客户信息、连接知识库搜案例、连接网页查企业背景
- 销售理解：就像公司的**供应商管理系统** — 标准化地对接各种外部资源

#### Agent = 公司里的专业岗位

| Agent 角色 | 真实公司对应 | 做什么 |
|-----------|-------------|-------|
| sales-master（销售主控） | 销售总监 | 接到任务后，分配给合适的人去做 |
| company-research（企业调研） | 市场调研专员 | 调研客户公司背景、行业信息 |
| bid-strategist（投标策略） | 商务投标专员 | 分析评分表、算分、控标 |
| meeting-analysis（会议分析） | 销售助理 | 分析会议纪要、提取关键信息 |

- 你不需要认识每个人，只需要跟**销售总监（sales-master）**说你的需求
- 销售总监会自动判断该派谁去做、按什么顺序做
- 这就是 **"统一入口、智能调度"**

#### Skill = 员工的专业技能

| Skill | 对应的职业技能 | 举例 |
|-------|-------------|------|
| 企业调研 | 调研分析能力 | "查一下招商银行的背景" |
| 案例匹配 | 信息检索能力 | "找找金融行业的 CMDB 案例" |
| 需求匹配 | 对比分析能力 | "这 50 条需求我们能满足多少" |
| 销售话术 | 沟通表达能力 | "帮我写一段拜访话术" |
| 投标控标 | 商务分析能力 | "这个评分表我们能拿多少分" |

- 一个 Agent 可以调用多个 Skill，就像一个员工可以有多种技能
- **关键区别**：传统 AI = 一个什么都会一点但什么都不精的通才；**Elevo Agent = 一个有专业知识、专业工具、专业流程的专家团队**

#### Plugin = 一个完整的部门

- sale-jerry-plugin = **整个销售支持部门**（1 个销售总监 + 32 项专业技能 + 知识库 + 外部数据源）
- 预售 Plugin = **售前技术支持部门**（技术方案、POC 测试、工作量评估）
- 安装一个插件 = 整个部门到岗

### Slide 10：动漫故事视角 — "勇者小队冒险记"

> **场景设定**：你是一位国王（销售），面对一头巨大的魔王（客户项目），你需要组建一支勇者小队去完成冒险。

#### 大模型 = 勇者的"灵魂之力"

> 每个勇者体内都蕴含着灵魂之力（大模型），决定了他们的智慧水平和战斗能力。

- **Claude Opus** = 传说级灵魂，能力最强，但消耗魔力也最大（适合打 Boss）
- **Claude Sonnet** = 精英级灵魂，能力均衡（日常冒险首选）
- **Claude Haiku** = 学徒级灵魂，快速敏捷（打小怪、跑腿）
- **灵魂可以更换** — 同一个勇者换上不同灵魂，能力就不同

#### MCP = 传送门与宝箱

> 世界各地散落着传送门和宝箱（MCP 通道），勇者们可以通过它们获取外部世界的力量和资源。

- **记忆宝箱**（claude-mem MCP）：打开后能看到之前所有冒险的记忆，不会重复犯错
- **万维传送门**（web-reader MCP）：能瞬移到任何网页获取信息
- **知识卷轴库**：存储了 843 个前辈勇者的冒险记录（案例库）

#### Agent = 勇者小队的成员

```
勇者小队（Agent 团队）
│
├── 队长・艾莉丝（sales-master）
│   "我是队长，告诉我你要做什么，我来安排！"
│   能力：识别任务类型 → 分配给合适的队员 → 监督执行 → 汇总成果
│
├── 侦察兵・洛克（company-research）
│   "我去侦查敌方情报！"（企业调研）
│   能力：搜索企业信息、分析行业、识别 IT 痛点
│
├── 军师・梅林（bid-strategist）
│   "让我来分析战局！"（投标策略）
│   能力：分析评分规则、计算得分、制定控标策略
│
├── 记录官・艾达（meeting-analysis）
│   "我会记录一切重要信息！"（会议分析）
│   能力：SPIN 分析、判断敌我态势（销售阶段）、生成推进建议
│
└── 训练师・兰斯（roleplay-dojo）
    "让我来模拟战斗！"（销售陪练）
    能力：扮演魔王（客户）、模拟对话、复盘分析
```

#### Skill = 每个勇者的专属技能树

```
侦察兵・洛克的技能树：
├── 🗺️ 企业调研 — 搜索目标企业背景信息
├── 🔍 案例匹配 — 从 843 个案例中找最相似的
├── 📝 销售话术 — 生成针对性的沟通话术
└── 📦 一站式准备 — 同时释放以上三个技能（组合技！）
```

- 每个勇者有自己的技能树，不同场景使用不同技能
- 队长（sales-master）会决定让谁用什么技能

#### Plugin = 勇者公会

- **sale-jerry-plugin** = "销售勇者公会"（1 个队长 + 16 个销售技能 + 12 个售前技能 + 4 个通用技能）
- 安装一个插件 = 整个公会入驻你的王国
- 你可以安装多个公会，让更多勇者为你服务

## 第四部分：CMDB 场景实战拆解（4-5 页）

### Slide 12：场景设定 — 某银行 CMDB 项目

> **背景**：你接到一个线索 — XX 银行要建设 CMDB 平台，你需要在 1 周内完成从客户调研到方案准备的全部工作。

#### 传统方式 vs Elevo 方式

| 步骤 | 传统方式 | Elevo 方式 | 效率提升 |
|------|---------|-----------|---------|
| 1. 客户调研 | 花半天查资料 | 输入"调研XX银行" → 5 分钟 | **96%** |
| 2. 找同行业案例 | 翻历史 PPT、问同事 | 自动匹配 843 个案例 → 2 分钟 | **99%** |
| 3. 需求匹配 | 逐条对比 Excel，2-3 天 | 上传需求文档 → 自动匹配 → 10 分钟 | **99%** |
| 4. 会议分析 | 凭记忆，经常遗漏 | 上传会议纪要 → SPIN 分析 → 5 分钟 | **95%** |
| 5. 投标准备 | 人工算分，1-2 天 | 上传评分表 → 自动算分控标 → 15 分钟 | **99%** |

### Slide 13：CMDB 场景 — 各层级角色拆解

> **场景**：销售输入"为 XX 银行-CMDB 项目准备销售资料"

#### 第 1 层：Elevo 平台 — 战场指挥部

- 提供工作空间（项目协作环境）
- 提供知识库（843 个案例、产品功能清单、资质库）
- 提供对话界面（自然语言交互入口）
- 提供成员权限管理（多人协作、数据隔离）
- **增值点**：一切都在一个平台上完成，不需要在 5 个工具之间来回切换

#### 第 2 层：大模型（LLM）— 思考引擎

- 理解销售的自然语言输入："为 XX 银行-CMDB 项目准备销售资料"
- 推理出需要执行的步骤：调研 → 匹配案例 → 生成话术
- 生成专业的分析报告和销售话术
- **增值点**：不是模板拼凑，而是真正理解业务语境后的智能生成

#### 第 3 层：MCP — 外部数据通道

| MCP 通道 | 在本场景中的作用 |
|---------|---------------|
| web-reader MCP | 搜索 XX 银行的公开信息（官网、年报、新闻） |
| claude-mem MCP | 搜索历史会话记忆，看之前是否做过银行 CMDB 项目 |
| 文件系统 MCP | 读取工作空间中的案例库文件、功能清单文件 |
| **增值点** | AI 不是凭空编造，而是基于**真实数据**（企业公开信息 + 内部知识库）生成分析 |

#### 第 4 层：Agent（智能体）— 销售总监

- **sales-master（销售主控）** 接收指令
- 自动识别阶段：这是一次"销售准备"任务
- 智能调度：
  1. 先调用 `company-research`（企业调研）
  2. 再调用 `case-matching`（案例匹配）
  3. 最后调用 `sales-script`（话术生成）
- 自动更新项目状态文件
- **增值点**：销售只需要说一句话，不需要知道有哪些工具、按什么顺序执行

#### 第 5 层：Skill — 具体执行者

| Skill 执行步骤 | 输入 | 输出 | 耗时 |
|--------------|------|------|------|
| company-research | "XX银行" | 企业背景报告（行业、规模、IT 架构） | 2 分钟 |

| project-status-updater | 更新项目进展 | 项目状态文件自动更新 | 自动 |
| **增值点** | — | **5 分钟完成过去半天的准备工作** | — |

### Slide 14：传统 CMDB 售前 vs AI 赋能后的 CMDB 售前

#### 传统方式的痛点

| 环节 | 痛点 | 原因 |
|------|------|------|
| 客户调研 | 信息不全、过时 | 手动搜索效率低，没有标准化流程 |
| 案例展示 | 找不到最相关的案例 | 案例散落在各处，靠记忆和运气 |
| 需求匹配 | 算不准满足率 | 人肉逐条对比，容易出错和遗漏 |
| 会议跟进 | 信息遗漏，判断不准 | 靠记忆，没有结构化分析方法 |
| 投标控标 | 算分慢，容易丢分 | 手动计算，经验依赖 |

#### AI 赋能后的改变

| 环节 | AI 赋能后 | 为什么 AI 能做到不一样的价值 |
|------|----------|--------------------------|
| 客户调研 | 5 分钟输出结构化报告 | AI 可以同时搜索多个信息源，自动提取关键信息 |
| 案例展示 | 从 843 个案例中智能匹配 | AI 可以同时分析行业、模块、规模等多个维度，找到最优匹配 |
| 需求匹配 | 10 分钟完成 50+ 条需求匹配 | AI 可以逐条精确匹配产品功能清单，计算满足率，标记定制项 |
| 会议跟进 | SPIN 分析 + 销售阶段判断 | AI 可以结构化分析对话内容，识别 S/P/I/N 四维度信息 |
| 投标控标 | 15 分钟完成算分 + 控标策略 | AI 可以同时计算多项得分、识别风险、生成防御/进攻策略 |

#### 为什么 AI 能实现不一样的价值输出？

1. **并发处理**：AI 可以同时做 10 件事，人一次只能做 1 件
2. **精确匹配**：AI 可以在几秒内遍历 843 个案例的所有字段，人做不到
3. **结构化分析**：AI 可以按 SPIN 等方法论逐条分析，人容易遗漏
4. **持续积累**：AI 记住所有历史案例和经验，新人也能立即拥有老销售的经验
5. **24 小时在线**：不需要等同事有空，随时随地可以使用

| **06** | **销售准备一站式** | `sale-jerry-plugin:sales-prep` | 一站式完成企业调研、案例匹配和话术生成的完整流程，自动生成销售准备资料包 | "为'中国银行'准备销售资料" |
| **07** | **销售话术生成** | `sale-jerry-plugin:sales-script` | 扮演20年经验的顶级销售专家，针对运维部门生成场景化开场、痛点挖掘、价值呈现、异议处理等完整销售话术 | "生成针对运维总监的拜访话术" |
| **08** | **技术黑话解码** | `sale-jerry-plugin:jargon-decoder` | 将客户技术术语（K8s、微服务、云原生）翻译成销售能懂的商业语言，并映射为优维标准产品功能 | "解释：客户说要做容器化和服务网格" |
| **09** | **会议纪要分析** | `sale-jerry-plugin:meeting-analysis` | 基于SPIN销售法分析会议内容，判断销售阶段并生成推进建议，自动生成会议分析报告和销售话术 | "分析上传的会议纪要" |
| **10** | **SPIN深度解读** | `sale-jerry-plugin:spin-analysis` | 基于SPIN方法论（现状/痛点/影响/需求）分析会议纪要或客户对话，识别四维度信息，判断销售阶段 | "用SPIN分析这段客户对话" |
| **11** | **需求精准匹配** | `sale-jerry-plugin:requirement-matching` | 解析客户需求文档（Excel/PDF），逐条匹配产品功能清单，计算满足率并标记定制项，基于真实功能清单 | "分析上传的需求文档" |
| **12** | **商务控标专家** | `sale-jerry-plugin:bid-strategist` | 分析招标商务评分表，识别废标风险，精确计算得分，生成防御性和进攻性控标建议，基于真实案例库和资质库 | "分析上传的商务评分表" |
| **13** | **销售实战陪练** | `sale-jerry-plugin:roleplay-dojo` | 基于真实项目数据进行销售场景演练，通过"侧写→演练→复盘"三阶段流程，模拟真实客户对话 | "针对'招商银行CMDB项目'进行销售演练" |
| **14** | **文档智能处理** | `sale-jerry-plugin:document-processor` | 支持Excel、Word、图片、Markdown、TXT等多种文件格式的读取、解析、提取和转换 | "提取这份PDF中的需求清单" |
| **15** | **FABE售前方案** | `sale-jerry-plugin:fabe-proposal` | 基于FABE方法论（特征→优势→利益→证据）生成正式售前方案，优先引用知识库产品功能清单，支持Markdown和Word输出 | "为CMDB项目生成FABE方案" |
| **16** | **AI需求可行性分析** | `sale-jerry-plugin:ai-feasibility-analysis` | 上传客户AI需求Excel文件，自动逐条评估落地可行性（三级分类），生成含Agent架构设计、分阶段计划、风险识别和投入产出预估的完整分析报告 | "分析这份AI需求清单的落地可行性" |
| **17** | **立项报告生成** | `sale-jerry-plugin:project-report` | 根据企业名称和模块名称，智能搜索知识库模板，自动生成定制化立项报告，支持CMDB、ITSM、自动化运维等各模块 | "生成CMDB立项报告" |

## 🔥 典型使用场景

### 场景1：新销售项目启动
```
用户："初始化客户项目'农夫山泉-CMDB项目'"
→ 自动创建项目目录结构
→ 生成项目状态.md和项目相关方.md模板
```

### 场景2：客户拜访准备
```
用户："为'招商银行'准备销售资料"
→ 自动调研企业信息
→ 匹配金融-银行行业的CMDB成功案例
→ 生成针对运维总监的销售话术
```

### 场景3：会议后跟进
```
用户：上传会议纪要文件
→ 自动SPIN分析（现状/痛点/影响/需求）
→ 判断销售阶段（需求探索/方案呈现/商务谈判）
→ 生成下一步推进建议和话术
```

### 场景4：售前技术支持
```
用户：上传招标文件
→ 自动分析招标文件（8章标准报告）
→ 生成投标技术方案（15-20万字）
→ 生成投标实施方案（3-5万字）
```

### 场景5：需求响应
```
用户：上传客户需求Excel
→ 逐条匹配产品功能清单
→ 计算满足率（标配/定制/不支持）
→ 标记定制化需求和工作量评估
```

### 场景6：AI需求可行性分析
```
用户：上传客户AI需求清单Excel + 样板文件
→ 自动读取并统计需求信息
→ 逐条三级可行性评估（可直接交付/Skill定制/深度定制）
→ 设计Agent/Skill架构方案
→ 输出分阶段落地计划与投入产出预估
```

### 场景7：投标应对
```
用户：上传招标商务评分表
→ 自动计算公司得分
→ 识别废标风险项
→ 生成控标建议（加分项/防御项）
```

### 场景8：POC测试管理
```
用户："生成POC测试用例"
→ 基于技术规范书分析需求
→ 生成结构化测试用例（Markdown + Word）
→ 执行测试后生成POC测试报告
```

### 场景9：FABE方案生成
```
用户："为CMDB项目生成FABE方案"
→ 输入客户问题和需求
→ 自动生成特征→优势→利益→证据结构化方案
→ 同时输出Markdown和Word格式
```

### 场景10：客户交流记录
```
用户：粘贴客户会议交流内容或上传文件
→ 自动生成标准化会议纪要/需求记录/Q&A文档
→ 保存到客户交流目录
```

## 🎓 最佳实践

### 销售流程
1. **项目启动时**：先用 `project-init` 创建标准目录
2. **客户拜访前**：用 `sales-prep` 一站式准备资料
3. **会议结束后**：立即用 `meeting-analysis` 分析纪要
4. **收到需求时**：用 `requirement-matching` 快速响应
5. **AI需求评估**：用 `AIFeasibility-analysis` 分析落地可行性
6. **方案输出**：用 `fabe-proposal` 生成FABE售前方案
7. **投标阶段**：用 `bid-strategist` 精准控标
8. **技术沟通**：用 `jargon-decoder` 理解客户术语
9. **实战演练**：用 `roleplay-dojo` 提升销售能力
10. **每日总结**：用 `daily-report` 生成工作日报

### 售前技术支持流程
1. **项目启动**：用 `presales-init` 创建售前目录
2. **收到招标文件**：用 `tender-analysis` 深度分析
3. **方案编写**：用 `solution-writing` 或 `bid-technical` 生成方案
4. **实施方案**：用 `bid-implementation` 生成实施方案
5. **POC准备**：用 `poc` 生成测试用例
6. **POC总结**：用 `poc` 生成测试报告
7. **客户交流**：用 `customer-comm` 生成标准化交流文档
8. **工作量评估**：用 `eva-workload` 评估项目工作量
9. **工作说明书**：用 `sow-writing` 编写SOW文档
